# Image Upload

사진과 이미지 산출물을 Cloudflare 기반 이미지 저장소에 업로드하고, 반환 URL을 검증해 필요한 문서나 워크플로우에 연결하는 스킬입니다.

## 기본 원칙

- 업로드 엔드포인트는 `https://upload.shdkej.com`입니다.
- 사용자가 준 원본 사진은 `https://upload.shdkej.com?kind=original`로 업로드합니다.
- 카드뉴스, 썸네일, 재렌더, 생성 이미지, 편집 이미지 같은 가공 산출물은 `https://upload.shdkej.com?kind=derived`로 업로드합니다.
- 요청은 이미지 파일의 MIME 타입에 맞는 `Content-Type`과 `Authorization: Bearer $CLOUDFLARE_IMAGE_UPLOAD_TOKEN`을 함께 보냅니다.
- 토큰은 로컬 env에서만 로드하고, 채팅·문서·로그·커밋에 절대 출력하지 않습니다.
- 런타임에서 토큰을 읽지 못할 때만 사용자에게 새 토큰 제공을 요청합니다.
- 업로드 후에는 반환 URL을 HTTP 요청으로 확인하고, 접근 가능할 때만 장기 보존 문서나 라이브러리에 삽입합니다.

## 작업 흐름

1. 이미지의 성격을 판정합니다.
   - 사용자가 직접 보낸 사진/스크린샷: `original`
   - AI 생성, 카드뉴스 렌더, 썸네일, 리사이즈/편집 결과: `derived`
   - 사용자가 목적지를 지정하면 그 목적지를 우선합니다.

2. 파일 위치와 MIME 타입을 확인합니다.
   - 로컬 파일이면 `file --mime-type` 등으로 타입을 확인합니다.
   - Telegram/OpenClaw 첨부 파일이면 먼저 접근 가능한 로컬 경로나 미디어 파일로 받은 뒤 업로드합니다.

3. 업로드합니다.
   - 토큰은 환경변수 `CLOUDFLARE_IMAGE_UPLOAD_TOKEN`에서 읽습니다.
   - `.env`를 로드해야 한다면 값 자체를 출력하지 않는 방식으로만 사용합니다.
   - 카드뉴스/썸네일/재렌더 산출물은 기본적으로 `?kind=derived`입니다.

4. 반환 URL을 검증합니다.
   - HTTP 200 또는 이미지로 접근 가능한 성공 응답을 확인합니다.
   - 실패하면 토큰, MIME 타입, 파일 경로, 엔드포인트 종류를 점검하고 재시도 여부를 판단합니다.

5. 결과를 연결합니다.
   - 문서 요청이면 반환 URL을 대상 문서에 삽입합니다.
   - 카드뉴스면 라이브러리 `items/index`나 템플릿 데이터에 최종 카드 URL을 반영합니다.
   - 단독 사진이면 `system/data/daily-tracking/YYYY-MM-DD.md`에 타임스탬프, 최소 맥락, URL을 남깁니다.

## 단독 사진 기본 처리

사용자가 사진만 보내고 별도 목적지나 분석 요청을 하지 않으면, 일일 트래킹 입력으로 처리합니다.

- `?kind=original`로 업로드합니다.
- 반환 URL을 검증합니다.
- `system/data/daily-tracking/YYYY-MM-DD.md`에 같은 타임스탬프 아래 기록합니다.
- 요청받지 않은 OCR, 감상, 분류, 분석은 하지 않습니다.

여러 장이 함께 도착하면 모든 이미지를 같은 흐름으로 처리하고, 같은 타임스탬프 묶음 아래에 기록합니다.

## 분석 요청이 있을 때

사용자가 이미지 분석, OCR, 요약, 분류, 일상 해석을 요청하면 업로드와 별개로 분석을 수행할 수 있습니다.

- 분석 결과에는 필요한 최소 맥락만 남깁니다.
- 사적인 이미지 내용은 사용자가 요구한 범위 밖으로 확장하지 않습니다.
- 장기 문서에는 원본 이미지 URL과 사용자가 요청한 분석 결과만 기록합니다.

## 카드뉴스와 생성 이미지

카드뉴스, 썸네일, 이미지 재렌더 작업에서는 최종 사용될 이미지만 업로드합니다.

- 내부 생성 중간 파일은 필요할 때만 보존하고, 공개 라이브러리에는 최종 렌더 URL을 연결합니다.
- `insight-card-maker`와 함께 사용할 때는 최종 카드 이미지를 `?kind=derived`로 업로드한 뒤, 업로드 URL 수와 카드 수가 일치하는지 확인합니다.
- 이미지 안에 글자/숫자/로고/워터마크가 없어야 하는 작업은 업로드 전 시각 확인을 우선합니다.

## 보고 형식

사용자에게는 업로드 완료 여부와 URL만 간결하게 보고합니다.

- 성공: 업로드한 이미지 수, 사용한 구분(`original`/`derived`), 검증 여부, 필요한 링크
- 실패: 어느 단계에서 막혔는지, 사용자가 제공해야 할 것(예: 토큰 접근 불가, 원본 파일 경로 없음)

토큰, 내부 인증 헤더, 불필요한 로컬 경로는 보고하지 않습니다.
