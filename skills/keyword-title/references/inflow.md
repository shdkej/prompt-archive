# Inflow 모드 — 시드 한 문장 → 파생 키워드 5 → 유입 제목 5 → 픽

> 가장 단순한 주력 워크플로. 사용자가 한 문장을 던지면 **검색·소비 데이터로 키워드를 발굴**하고,
> 그 키워드로 **유입(검색 노출+클릭) 잘 될 제목 5개**를 추천한다. 사용자는 픽만 한다.
> 무거운 분석(만다라트·사분면·PersonalDepth)은 쓰지 않는다 — 발굴과 후킹에 집중.

---

## 흐름 (2단계)

```
[입력] 시드 한 문장 (예: "신혼여행으로 1년 세계여행 다녀온 이야기")

[1] 파생 키워드 5개 발굴 (실측 우선)
    ├── scripts/searchad_keyword.py  → related_top: 네이버 검색량 큰 연관어
    ├── scripts/youtube_keyword.py   → 시드+연관어의 유튜브 median 조회수
    ├── LLM 파생                      → API가 못 잡는 의미축 보강 (감정·상황·니치)
    ├── 클러스터링: 의미 겹치는 것 묶어 대표 1개
    └── 상위 5개 선정 (선정 점수 = 검색량/조회수 + 시드 연관성)
        → 각 키워드에 [네이버 검색량] / [유튜브 median] 나란히 표기

[2] 유입 제목 5개 생성
    ├── 위 5개 키워드를 재료로 (제목마다 검색 키워드 1개+ 박기)
    ├── 후킹: signals-youtube §CuriosityGap (결과 숨김) 적용
    ├── 플랫폼 태깅: 각 제목이 강한 곳 표시 (🔍네이버 / ▶️유튜브 / 공통)
    │   └── 플랫폼 역전 주의: 검색량 키워드와 조회수 키워드가 다를 수 있음
    └── 톤: 네이버형=담백(tone.md §블로그), 유튜브형=후킹(tone.md §YouTube)

[출력] 키워드 5 + 제목 5 → 사용자 픽
```

---

## 키워드 5개 선정 기준

1. **실측 데이터 우선**: 검색광고 `related_top`의 월 검색량 + YouTube `median_view`가 큰 것.
2. **시드 연관성**: 시드 문장의 핵심 의미에서 벗어나지 않을 것 (검색량 커도 무관하면 제외).
3. **다양성**: 5개가 서로 다른 각도 (비용·감정·지역·방법·결과 등). 같은 의미 중복 금지.
4. **fallback**: API 키 없으면 LLM이 의미론적으로 파생하되, "검색량 미확인" 명시.

## 유입 제목 5개 생성 규칙

- 각 제목은 **검색 키워드(Searchability) 1개 이상** + **호기심 갭(CuriosityGap)** 둘 다 갖춘다.
- 5개를 서로 다른 유형으로: 호기심갭형 / 숫자대비형 / 변화형 / 질문형 / 정보형.
- 거짓 낚시·공포 마케팅 금지 (tone.md §2, signals-youtube §5).
- 플랫폼 태그로 "이 제목은 어디서 유입 강함"을 표시 → 사용자가 채널 맞춰 픽.

## 출력 포맷

```markdown
# Inflow — {시드 요약 15자}

## 시드
{원문 한 문장}

## 1단계 · 파생 키워드 5개
| # | 키워드 | 네이버 검색량/월 | 유튜브 median | 각도 | 출처 |
|---|---|---|---|---|---|
| 1 | ... | 56,140 | 120k | 비용 | 검색광고 |
| ... |
> 데이터 출처: 검색광고 API / YouTube API (키 없으면 "LLM 추정" 표기)
> 플랫폼 역전 메모: {네이버 강세 키워드 vs 유튜브 강세 키워드가 다르면 한 줄}

## 2단계 · 유입 제목 5개 (픽하세요)
| # | 제목 | 유형 | 핵심 키워드 | 강한 플랫폼 |
|---|---|---|---|---|
| 1 | ... | 호기심갭 | 크루즈여행 | 🔍▶️ 공통 |
| 2 | ... | 숫자대비 | ... | ▶️ 유튜브 |
| ... |

## 다음
- 픽한 번호를 알려주면 그 제목 기준으로 변주 3개 더 / 썸네일 카피 제안.
```

---

## 측정

- 픽한 제목과 실제 유입(조회수·클릭·검색노출)을 `outputs/{slug}.md`에 기록.
- 어떤 "각도"의 키워드가 실제 유입으로 이어지는지 누적 → 선정 기준 개선.
