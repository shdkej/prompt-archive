# 깊이 × 희소성 매트릭스

> 출처: `Content_Strategy.md` "지식 깊이 × 희소성 매트릭스"
> 이 매트릭스는 키워드 후보를 4개 사분면으로 분류해 발행 우선순위를 정한다.

---

## 매트릭스

```
              희소성 높음
                  │
    ★ 최우선      │    ◆ 좋은 후보
    (깊이+희소)    │    (트렌드+희소)
                  │
──────────────────┼──────────────────
                  │
    ○ 시리즈화     │    △ 선택적
    (깊이+경쟁)    │    (학습중)
                  │
              희소성 낮음

깊이 높음 ────────────────── 깊이 낮음
```

---

## 사분면 정의 + 처리 가이드

### ★ 최우선 — 깊이 + 희소
- **조건**: 내가 직접 겪은 실전 경험 + 한국어 시장에서 비슷한 깊이의 글이 드묾
- **예시**: "EKS 마이그레이션 1.20→1.27 실전기", "분당 5만건 메시지 처리 최적화"
- **출력 액션**: 블로그 본글 제목 후보를 우선 생성. Threads에 가설 시드 3개 함께 제안.
- **발행 우선순위**: 1순위

### ◆ 좋은 후보 — 트렌드 + 희소
- **조건**: 내가 중급이지만 실전 적용 사례 자체가 드물어서 지금 쓰면 선점 가능
- **예시**: "Claude Code + MCP 워크플로우", "멀티 에이전트로 PA Flow"
- **출력 액션**: 블로그 시리즈 첫 글 제목 + Threads에서 가설 검증 시드.
- **발행 우선순위**: 2순위 (선점 가치 있음)

### ○ 시리즈화 — 깊이 + 경쟁
- **조건**: 내 경험은 깊은데 이미 좋은 글이 시장에 많음
- **예시**: "Clean Architecture 실전", "DevOps는 포지션이 아니라 문화"
- **출력 액션**: "내 경험이라는 차별점"을 강조하는 시리즈형 제목. 단발 글로는 비추.
- **발행 우선순위**: 3순위 (시리즈로 묶을 때만)

### △ 선택적 — 학습중 + 경쟁
- **조건**: 깊이도 얕고 경쟁도 많음
- **출력 액션**: Threads에 인사이트 조각으로만. 블로그 글은 만들지 않음.
- **발행 우선순위**: 4순위 (Threads 가설용)

## 사분면 판단 알고리즘 (LLM 프롬프트용)

```
input: 키워드 후보 1개, 시드 텍스트, 내 만다라트 8축 정보 (선택)

1. 깊이 점수 (0.0 ~ 1.0):
   - 시드 텍스트에 1인칭 실전 경험이 묻어있는가?
   - 구체적 수치/도구/날짜가 있는가?
   - 7년차 DevOps/백엔드 + 이커머스 MAU 300만 + AX 자동화 맥락에 부합하는가?
   - 만다라트 1~4축(Infra/DevOps/Architecture/Blogging)이면 가중치 +0.2
   → 0.7 이상이면 "깊이 높음"

2. 희소성 점수 (0.0 ~ 1.0):
   - 한국어로 같은 깊이의 글이 5개 이상 검색되는가?
   - 영어권에는 있지만 한국어로는 부족한가?
   - LLM 학습 데이터에 비슷한 글이 자주 등장했는가?
   → 0.6 이상이면 "희소성 높음"

3. 사분면 매핑:
   - 깊이 ≥ 0.7 + 희소 ≥ 0.6 → ★ 최우선
   - 깊이 < 0.7 + 희소 ≥ 0.6 → ◆ 좋은 후보
   - 깊이 ≥ 0.7 + 희소 < 0.6 → ○ 시리즈화
   - 깊이 < 0.7 + 희소 < 0.6 → △ 선택적
```

## 만다라트 8축 (참고)

키워드가 어느 축에 속하는지 표기하면 빈 영역 추적에 도움이 된다.

| # | 축 | 비고 |
|---|---|---|
| 1 | Infra/Architecture | 깊이 많음 (이미 글 다수) |
| 2 | DevOps/Monitoring/AI | ★ 최우선 영역이 가장 많음 |
| 3 | Integration/Business/Work | **빈 영역 (Threads 가설부터)** |
| 4 | Blogging/Note | ◆ 좋은 후보 다수 |
| 5 | Human (성장/밸런스) | **빈 영역** |
| 6 | Health (개발자 루틴) | **빈 영역** |
| 7 | Idea (여행/영화 인사이트) | **빈 영역** |
| 8 | (TBD) | - |
