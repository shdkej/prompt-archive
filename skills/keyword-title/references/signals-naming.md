# Naming Signals (앱/제품/서비스 이름용)

> `signals-content.md`는 **블로그·Threads 제목 평가용**.
> 이 문서는 **앱·제품·서비스 이름 평가용**. 평가축 자체가 다르다.
> 두 신호를 섞지 말 것.

---

## 1. 왜 별도인가

| 콘텐츠 제목 | 제품 이름 |
|---|---|
| 검색·바이럴이 핵심 → Velocity·Volume 중요 | 검색·바이럴은 부차적, **기억성·발음** 중요 |
| 1회용 (글 1편의 후크) | 영구적 (앱이 살아있는 동안 매일 호출됨) |
| 70자 이내, 정보 함량 우선 | 1~6자, 정보보다 **공기 함량** 우선 |
| 후크가 자극적이어도 됨 | 도메인·법무·SNS 핸들 충돌 회피 필요 |

---

## 2. 신호 6종

| 신호 | 의미 | 측정 | 가중치 |
|---|---|---|---|
| **Memorability** | 한 번 듣고 다시 떠올릴 수 있나 | 음절수(1~4 최적) · 발음 용이성 · 시각화 가능성 | 0.25 |
| **Concept Density** | 한 단어에 핵심 메타포가 얼마나 압축됐나 | 단어 의미와 제품 본질의 거리 (가까울수록 높음) | 0.20 |
| **Tonal Fit** | BRAND 톤(따뜻·미니멀·옆자리선배)에 부합 | tone.md 6항목 + 차가운 기업 톤 회피 | 0.20 |
| **Differentiation** | 같은 카테고리 다른 제품과 헷갈리지 않나 | 동명/유사명 앱 검색 결과 + 일반명사화 위험 | 0.15 |
| **Extensibility** | 카피·서브브랜드·도메인으로 변주 가능 | 짧은 카피 3개 만들어보기 / 도메인 후보 3개 / SNS 핸들 | 0.10 |
| **Wit Layer** | 다중 의미·말맛·발견의 재미 | 이중 의미 / 의외성 / 컨셉과의 유머 정합 | 0.10 |

---

## 3. 점수 산식

```
Naming Score (0~100) =
  100 × (
    0.25 · Memorability
  + 0.20 · ConceptDensity
  + 0.20 · TonalFit
  + 0.15 · Differentiation
  + 0.10 · Extensibility
  + 0.10 · WitLayer
  )

각 신호는 0.0 ~ 1.0
```

### 컷오프
- **75점 이상**: 최종 후보로 출력
- **60~74점**: 출력하되 "변주 필요" 메모 표기
- **60점 미만**: 출력 목록에서 제외

---

## 4. 각 신호의 평가 가이드

### Memorability (가장 중요한 신호)
- 음절수: 2~4음절 = 1.0 / 1음절 = 0.7 / 5음절 = 0.7 / 6음절+ = 0.4
- 발음 충돌(된소리·외래어 자음군)이 있나 → 있으면 -0.2
- 시각화 가능: 이름 들으면 이미지가 떠오르는가 → 그렇다 +0.1
- 한국어/영어 어느 쪽으로 읽어도 똑같이 들리나 → 그렇다 +0.1

### Concept Density
- 제품 본질을 직접 표현 (예: 일기 앱 → "다이어리"): 0.4 (직설은 밀도 낮음 — 일반명사화 함정)
- 본질을 메타포로 표현 (예: 일기 앱 → "수첩"): 0.8
- 본질과 무관한 추상어/조어: 0.3
- **중간 강도 메타포**가 최고점 (직설 > 가까움 > 멀어짐 순서 X — U자 곡선)

### Tonal Fit
- tone.md 6항목 통과: 기본 0.6
- "따뜻함" 1차 인상이 느껴지나 → +0.2
- 차가운 기업 톤 단어 포함 (Pro·Plus·X·Tech·Hub) → -0.4
- 유행어/줄임말 의존도 → 너무 높으면 -0.2 (3년 후 촌스러워질 위험)

### Differentiation
- 앱스토어/Play 스토어 검색 충돌: 동명 0개 = 1.0 / 1~2개 = 0.7 / 3~5개 = 0.4 / 5+ = 0.2
- 일반명사화 위험: "다이어리", "노트", "메모"처럼 SEO 자체가 무의미해지면 -0.3
- 브랜드 검색 가능성: 구글에 이름만 쳤을 때 1페이지에 우리 앱이 뜰 가능성

### Extensibility
- 도메인: `.app`, `.com`, `.co.kr` 중 2개 이상 가용 → +0.3
- SNS 핸들: Instagram/Threads에서 사용 가능 → +0.2
- 카피 변주: "X로 시작하기" "X하는 사람들" 류 카피 3개 만들어지나 → +0.3
- 서브브랜드: 향후 "X Pro"가 아니라 자연스러운 라인업 확장 가능 → +0.2

### Wit Layer
- 이중 의미가 컨셉과 자연스럽게 닿는다: 1.0
  - 예: 환생 게임 → "다음 생" (직역이지만 다른 일정 의미로도 읽힘)
- 들었을 때 한 박자 후에 의미가 풀리는 발견의 재미: 1.0
- 단순히 귀엽기만 함: 0.5
- 위트 없음 / 의도 불분명: 0.2

## 5. 자동 폐기 조건

다음 중 하나라도 해당하면 점수와 무관하게 제거:

- BRAND 안티 포지셔닝 단어 포함: "Pro", "X", "AI" (서비스 본질이 AI라도 이름에 박지 않음 — BRAND 톤이 차가워짐)
- 한국에서 발음 시 부정 의미 (욕설/속어/사투리 부정형)
- 영어로 읽었을 때 부정 의미 (네이밍 글로벌화 가능성 봤을 때)
- 등록상표 충돌 가능 (LLM 자가 판단 — "이거 기업명 같은데"라는 의심)
- 음절수 7개 이상 (앱 아이콘 아래 자르지 않고 표기 어려움)

---

## 6. naming-categories.md와의 관계

평가 전에 후보를 **카테고리별로 분산 생성**한다 (`naming-categories.md` 10개 카테고리).
같은 카테고리에 후보가 몰리면 다양성 점수가 깎인다.

출력 시 5~7개 최종 후보가 **최소 4개 이상의 카테고리에서** 나와야 한다.
