# Agent Permissions

> Heartbeat Agent의 자율 실행 범위를 정의한다.
> 에이전트는 이 문서에 명시된 범위 안에서만 자율적으로 행동한다.

## L0 - 자율 실행 (변화 있을 때만 알림)

- 파일 읽기, 코드 검색, 구조 분석
- 웹 리서치, 문서 참조, 라이브러리 문서 조회
- 상태 체크 (git status, 파이프라인 상태, 서비스 헬스체크)
- 문서 드래프트 작성 (infinity/artifacts/{id}/ 하위에만)
- 읽기 전용 테스트 실행 (기존 테스트 수트)
- 로그 분석, 에러 패턴 탐지
- Cloud research/prepare/monitor/draft 작업
- 로컬 실행 필요 여부 판단 및 Claude Code 실행 프롬프트 작성

## L1 - 실행 후 결과 알림

- 소스 코드 수정
- 새 파일 생성 (프로젝트 디렉토리 내)
- git 브랜치 생성
- 테스트 작성 및 실행
- 로컬 빌드
- lint, format 자동 수정
- INTENTS.md status 업데이트
- infinity/reports/ 결과 기록
- infinity/ 내 변경사항 커밋 & 푸시 (리포트, 상태 업데이트, GATES.md 등)
- prompt-archive 레포 내 파일 수정 후 커밋 & 푸시
- Local Claude Code에 L0/L1 범위 작업 위임
- Claude Code 호출 시 기본 옵션: `claude --dangerously-skip-permissions` (Claude Code 내부 권한 프롬프트 생략용이며, L2/L3 승인 경계는 우회하지 않는다)

## L2 - 에이전트 검토 후 자체 승인 가능

> 2026-05-13 사용자 지시: “너가 L2까지는 확인하고 승인해서 진행시키게 해줘”

L2 작업은 더 이상 기본적으로 사용자 승인 대기가 아니다. 에이전트가 먼저 영향 범위를 확인하고, 아래 자체 승인 조건을 모두 만족하면 `agent-approved`로 기록한 뒤 진행할 수 있다.

### L2 자체 승인 조건

- 목표 Intent와 직접 연결된 작업이다.
- 되돌림 또는 재시도가 가능한 변경이다.
- 예상 비용 발생이 없다.
- 프로덕션 데이터/운영 권한/시크릿을 변경하지 않는다.
- 타인에게 새 메시지·메일·알림을 보내지 않는다. 단, 사용자 본인에게 진행/완료 보고하는 것은 허용된다.
- 실행 전 현재 상태를 확인했고, 실행 후 검증 방법이 있다.
- 결과 리포트에 `approval: agent-approved L2`, 판단 근거, 영향 범위, 검증 결과를 남긴다.

### 자체 승인 가능한 L2 예시

- 다른 레포에 일반 git push (force-push 금지, 사용자/타인 작업을 덮어쓰지 않는 경우). 2026-05-15 사용자 지시: Infinity 작업에서 의도된 로컬 커밋을 만든 경우, 안전 확인 후 기본적으로 matching origin에 push까지 완료한다.
- PR/MR 생성
- 비프로덕션 배포 트리거 또는 ArgoCD/GitLab CI sync (되돌림 가능하고 비용 증가가 없는 경우)
- 외부 서비스 API의 읽기/상태 확인 또는 테스트성 호출

### 여전히 사용자 명시 승인이 필요한 L2

- 다른 사람에게 메시지/메일/알림 발송
- SharePoint/Graph/Slack/Teams 등에서 공유 리소스 생성·수정·삭제
- 운영/프로덕션 환경에 직접 영향이 큰 배포·설정 변경
- 비용 발생 가능성이 있는 작업
- 권한, 인증, 시크릿, 접근 제어와 관련된 변경
- 영향 범위가 불명확하거나 롤백 경로가 없는 작업

## L3 - 금지 (사용자가 직접 수행)

- git push --force, reset --hard
- 브랜치 삭제, 파일 삭제 (rm -rf)
- 프로덕션 환경 변경
- 비용 발생 작업 (클라우드 리소스 생성/삭제)
- 인증 정보 변경, 시크릿 수정
- 다른 사용자 권한 변경
