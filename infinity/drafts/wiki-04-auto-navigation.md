# wiki-04: agent-wiki 전체 문서 자동 네비게이션

> Intent: wiki-04 | 작성: 2026-04-24T09:00 | L2 승인 대기 (agent-wiki push)

## 문제 분석

### 현재 상황
- wiki-02: Docsify + GitHub Pages 구축 (`loadSidebar: true`)
- wiki-03: 모바일 토글/오버레이/자동닫힘 UX 구현
- `_sidebar.md`: 수동으로 유지되는 하드코딩 메뉴
  - 새 문서 추가 시 수동으로 `_sidebar.md`도 업데이트해야 함
  - 누락된 문서가 모바일에서 탐색 불가

### 요구사항
- 전체 문서 구조를 자동으로 반영하는 사이드바
- 모바일에서도 모든 문서 탐색 가능
- 새 문서 추가 시 수동 작업 없이 자동 업데이트

## 솔루션 선택: GitHub Actions + 자동 생성 스크립트

### 비교 검토

| 방식 | 장점 | 단점 | 선택 |
|------|------|------|------|
| A. docsify-auto-sidebar 플러그인 | 클라이언트 사이드, 설정 간단 | GitHub API rate limit, 파일 구조 노출 제한 | 보류 |
| **B. GitHub Actions 자동 생성** | 항상 정확, 빌드 타임 처리, 속도 빠름 | 워크플로 설정 필요 | **채택** |
| C. 수동 유지 | 제어 가능 | 현재 문제의 원인 | 기각 |

**채택 근거**: Docsify가 이미 `loadSidebar: true`로 `_sidebar.md`를 사용 중.
GitHub Actions으로 `_sidebar.md`를 자동 생성하면 기존 설정 변경 없이 해결 가능.

## 구현 파일

### 1. `.github/workflows/auto-sidebar.yml`

```yaml
name: Auto-generate _sidebar.md

on:
  push:
    branches: [main]
    paths:
      - '**/*.md'
      - '!_sidebar.md'
      - 'scripts/generate_sidebar.py'
  workflow_dispatch:

permissions:
  contents: write

jobs:
  generate-sidebar:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Generate _sidebar.md
        run: python3 scripts/generate_sidebar.py > _sidebar.md

      - name: Commit and push if changed
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add _sidebar.md
          git diff --staged --quiet || (
            git commit -m "chore: auto-update _sidebar.md [skip ci]"
            git push
          )
```

### 2. `scripts/generate_sidebar.py`

```python
#!/usr/bin/env python3
"""agent-wiki _sidebar.md 자동 생성기"""

import os
import re

EXCLUDE_FILES = {'_sidebar.md', '_navbar.md', '.nojekyll'}
EXCLUDE_DIRS = {'.git', '.github', 'scripts', 'node_modules'}

def get_title(filepath):
    """파일에서 # 헤딩 제목 추출, 없으면 파일명 사용"""
    try:
        with open(filepath, encoding='utf-8') as f:
            for line in f:
                line = line.strip()
                if line.startswith('# '):
                    return line[2:].strip()
    except Exception:
        pass
    return os.path.splitext(os.path.basename(filepath))[0]

def scan_dir(root, base='.', indent=0):
    lines = []
    try:
        entries = sorted(os.scandir(root), key=lambda e: (e.is_file(), e.name.lower()))
    except PermissionError:
        return lines

    for entry in entries:
        if entry.name.startswith('.') or entry.name.startswith('_'):
            continue
        if entry.is_dir() and entry.name not in EXCLUDE_DIRS:
            sub_lines = scan_dir(entry.path, base, indent + 1)
            if sub_lines:
                lines.append(f"{'  ' * indent}- **{entry.name}**")
                lines.extend(sub_lines)
        elif entry.is_file() and entry.name.endswith('.md'):
            if entry.name in EXCLUDE_FILES:
                continue
            title = get_title(entry.path)
            rel_path = os.path.relpath(entry.path, base).replace('\\', '/')
            # README.md는 디렉토리 링크로 표시
            if entry.name == 'README.md':
                dir_path = os.path.relpath(root, base).replace('\\', '/')
                if dir_path == '.':
                    continue  # 루트 README는 홈으로
                lines.insert(0, f"{'  ' * indent}- [{title}]({dir_path}/)")
            else:
                lines.append(f"{'  ' * indent}- [{title}]({rel_path})")
    return lines

def main():
    base = os.getcwd()
    print("<!-- Auto-generated by GitHub Actions — do not edit manually -->")
    print("- [Home](/)")
    print()
    for line in scan_dir(base, base, 0):
        print(line)

if __name__ == '__main__':
    main()
```

## 주의사항

- `[skip ci]` 태그로 자동 생성 커밋이 무한 루프 트리거 방지
- `paths: '!_sidebar.md'` 조건으로도 이중 방지
- `permissions: contents: write` 필요 (워크플로가 커밋하려면)
- 디렉토리가 비어있으면 섹션 헤더 미출력 (sub_lines 체크)
