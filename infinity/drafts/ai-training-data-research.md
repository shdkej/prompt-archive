## 1. 회사별 학습 데이터 유형 정리

### 1.1 Google (Gemini 계열)

**공개 근거**: Gemini 기술 보고서, Google 공식 발표

| 데이터 유형 | 내용 |
|---|---|
| 웹 크롤링 | 자체 크롤러로 수집한 공개 웹 문서 (`robots.txt` 준수) |
| 라이선스 데이터 | 상업적 라이선스 계약을 통해 획득한 데이터 |
| Google 사용자 데이터 | Google 제품/서비스 사용자 데이터 (AI 학습 동의한 경우) |
| 사용자 상호작용 | Gemini와의 대화 데이터 |
| 내부 생성 데이터 | Google이 업무 과정에서 획득·생성한 데이터 |
| 멀티모달 | 텍스트, 코드, 이미지, 오디오(음성 포함), 비디오 |

**데이터 처리**: 중복 제거, `robots.txt` 준수, 안전 필터링, 품질 필터링 (휴리스틱 규칙 + 모델 기반 분류기)

**특이점**: Google은 검색 엔진, YouTube, Google Books 등 방대한 자체 데이터 인프라를 보유. 이 독점 데이터 접근이 경쟁사 대비 큰 차별점.

---

### 1.2 OpenAI (GPT 계열)

**공개 근거**: GPT-3 논문, GPT-4 기술 보고서 (GPT-4는 구체적 데이터셋 구성 비공개)

| 데이터 유형 | 내용 |
|---|---|
| Common Crawl | GPT-3 기준 전체 토큰의 80%+ 차지. GPT-4도 사용 추정 |
| Wikipedia | 고품질 사실 기반 텍스트 |
| Books1 / Books2 | 광범위한 도서 데이터셋 (저작권 논란 있음) |
| 코드 | GitHub 등 공개 코드 저장소 (Codex 훈련 이후 강화) |
| 라이선스 데이터 | Axel Springer 등 미디어사와 상업적 계약 |
| 사용자 상호작용 | ChatGPT 사용자 피드백 데이터 |
| RLHF 데이터 | 인간 평가자(데이터 라벨러)의 선호도 학습 |

**데이터 처리**: GPT-4 기술 보고서는 "데이터셋 구성, 학습 방법 등에 대한 세부 정보를 포함하지 않는다"고 명시. 경쟁 이유로 공개 최소화.

**특이점**: ChatGPT의 엄청난 사용자 베이스(수억 명)를 통한 지속적 피드백 데이터 수집이 가능. 이 규모의 사용자 상호작용 데이터는 경쟁사가 따라가기 어려운 장점.

### 1.3 Anthropic (Claude 계열)

**공개 근거**: Claude 3 모델 카드, Constitutional AI 논문, Anthropic 투명성 보고서

| 데이터 유형 | 내용 |
|---|---|
| 공개 인터넷 데이터 | 자체 크롤러로 수집 (robots.txt 준수, 2025년 3월 기준) |
| 서드파티 비공개 데이터 | 외부 기업과 계약을 통한 비공개 데이터 |
| 데이터 라벨링 서비스 | 유료 계약자/라벨링 서비스 제공 데이터 |
| 사용자 데이터 | Claude 사용자 중 학습 동의한 사람의 대화 |
| 내부 생성 데이터 | Anthropic 내부에서 생성한 데이터 |
| Constitutional AI 합성 데이터 | AI가 헌법(Constitution)에 따라 자체 생성한 합성 데이터 |

**Constitutional AI (CAI)**:
- RLHF의 대규모 초기 적용 사례로 기록됨
- 2단계 프로세스: (1) 지도학습 - 모델이 헌법 기준으로 자체 응답 비판·수정 → (2) 강화학습 - AI 피드백으로 선호도 모델 학습
- 헌법 출처: UN 세계인권선언, 내부 상호작용 경험 등

**데이터 안전 필터링**: 2025년 발표에 따르면, 사전학습 데이터 필터링을 위한 분류기를 개발하여 유해 콘텐츠를 사전학습 데이터에서 제거 후 처음부터 재훈련.

**특이점**: 세 회사 중 안전성 중심의 데이터 처리에 가장 투명. CAI를 통한 합성 데이터 생성이 독보적 접근법.

## 2. 학습 데이터 차이가 도메인 강점에 미치는 영향

### 2.1 코딩 능력

| 회사 | 강점 요인 | 현재 수준 (2026 기준) |
|---|---|---|
| **Anthropic Claude** | 코드 품질 중심 학습, CAI로 정확한 추론 강화 | SWE-bench Verified **80.8%** (Claude Opus 4.6) - 1위 |
| **OpenAI GPT** | GitHub 코드 + ChatGPT 사용자 코딩 피드백 대규모 수집 | SWE-bench Verified **80%** (GPT-5.2) |
| **Google Gemini** | Google 내부 코드 데이터 + 알고리즘 집중 | 경쟁 프로그래밍 Elo **1500+** 돌파 (최초) |

**분석**: 실제 소프트웨어 엔지니어링(SWE-bench)에서는 Claude가 우세, 알고리즘 코딩 경쟁에서는 Gemini가 우세. 이는 학습 데이터에 포함된 코드의 "유형" 차이를 반영.

### 2.2 수학/논리 추론

- **Google Gemini 2.5 Pro**: 수학 벤치마크에서 강세 (추정: Google 내부 수학 데이터셋 + 알고리즘 데이터)
- **OpenAI GPT-4 o-시리즈**: Chain-of-Thought 추론 강화, 수학 올림피아드 수준
- **Anthropic Claude**: Constitutional AI의 단계별 추론 방식이 수학적 사고에 긍정적

### 2.3 다국어 지원

- **Google**: 검색 엔진·Gmail·번역 서비스의 다국어 데이터 독점 접근 → 다국어 강세
- **OpenAI**: Common Crawl의 다국어 데이터 + 전 세계 ChatGPT 사용자 피드백
- **Anthropic**: 영어 중심 안전 데이터 집중 (추정: 다국어 상대적으로 약함)

### 2.4 사실 정확도

- **Google**: 실시간 검색 통합(Grounding) + 구조화된 지식 데이터 접근
- **OpenAI**: 대규모 웹 크롤링 + 위키피디아, 하지만 학습 컷오프 한계
- **Anthropic**: 사전학습 데이터 안전 필터링이 오히려 사실 정확도에 긍정적 영향 (추정)

### 2.5 창의적 글쓰기

- **OpenAI GPT**: 방대한 Books1/Books2 소설·문학 데이터 → 스타일 다양성 강세
- **Anthropic Claude**: Constitutional AI로 가치관 일관성 있는 창의적 표현 (추정)
- **Google Gemini**: 상대적으로 창의적 글쓰기보다 사실 기반 생성에 최적화 (추정)

```
인터넷 전체 데이터 분포 (추정):
```

## 5. 결론

세 회사의 학습 데이터 전략은 각사의 강점과 철학을 반영합니다:

- **Google**: 방대한 자체 생태계 데이터 + 멀티모달 → 검색/다국어/알고리즘 강세
- **OpenAI**: 대규모 공개 웹 + RLHF 반복 개선 → 범용성·코딩·창의적 글쓰기 균형
- **Anthropic**: 안전 필터링 + CAI 합성 데이터 → 코드 품질·추론 정확성·안전성 강세

코딩 영역에서는 2026년 현재 세 회사 모두 SWE-bench 80% 내외로 수렴하고 있으며, 실제 차별화 요인은 벤치마크보다 특정 유형(알고리즘 vs. 실용 엔지니어링)에서 나타납니다.
