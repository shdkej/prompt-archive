## 3. 코딩 영역 강점 분석

### 벤치마크 결과 (2026년 기준)

✅ **공개 근거**: SWE-bench Verified (실제 GitHub 이슈 500개 기반)

| 모델 | SWE-bench Verified | 특화 강점 |
|------|-------------------|----------|
| Claude Opus 4.6 | **80.8%** (1위) | 대형 코드베이스, 긴 컨텍스트(1M 토큰) |
| Claude 4 Sonnet | **77.2%** | 일반 코딩 작업 |
| GPT-5 | **74.9%** | FastAPI, Django, 데이터 과학 라이브러리 |
| GPT-5.3-Codex | 1위 (Terminal-Bench 2.0 77.3%) | DevOps, 터미널 작업 |
| Gemini 2.5 Pro | **73.1%** | 추상적 추론, 멀티모달 |

### 코딩 강점 차이의 원인 추정

#### Claude가 코딩에서 1위인 이유 (⚠️ 일부 추정 포함)

1. **Constitutional AI의 정밀성**: CAI 방식이 "정확한 답변"을 중시하는 원칙을 내재화하여 코드 정확성에 유리
2. **긴 컨텍스트 처리**: 1M 토큰 컨텍스트 창으로 대규모 코드베이스 분석에 유리
3. **코드 데이터 품질**: ⚠️ 추정 — 고품질 코드 데이터 필터링에 집중한 것으로 보임

#### GPT가 DevOps/터미널에서 강한 이유 (⚠️ 추정)

1. **Microsoft 생태계**: Azure DevOps, GitHub Copilot과의 연동으로 실제 개발 워크플로우 데이터 풍부
2. **Codex 연구**: 코드 전용 모델(Codex) 연구를 통해 축적된 코드 학습 방법론
3. **합성 교과서 데이터**: 프로그래밍 교재 수준의 합성 데이터가 실용적 코딩 능력 향상에 기여

#### Gemini가 추론에서 강한 이유 (⚠️ 추정)

1. **멀티모달 설계**: 처음부터 다양한 모달리티를 함께 처리하는 훈련이 추상적 추론 능력 강화
2. **구글 내부 데이터**: ⚠️ 추정 — 수학 문제, 과학 문서 등 구글 서비스 데이터 활용
3. **MoE 아키텍처**: 전문가 모듈이 특정 유형의 문제에 집중하여 처리 효율 향상

## 4. 인터넷상 주제별 강점 추론

### 코딩 / 프로그래밍

- **Claude > GPT ≈ Gemini**: SWE-bench 기준, Claude가 전반적으로 가장 강함
- **GPT 특화**: DevOps, 터미널, Microsoft 스택(Azure, C#, PowerShell)
- **Gemini 특화**: 멀티모달 코딩(이미지+코드), 긴 컨텍스트 처리

### 창의적 글쓰기 / 대화

- **Claude > GPT > Gemini**: ⚠️ 추정 — Claude가 Constitutional AI로 인해 문체가 일관되고 창의성이 높다는 평가
- **공통 약점**: 특정 스타일 유지, 매우 긴 소설 구조 유지

### 수학 / 추론

- **Gemini 2.5 Pro ≈ o3 > Claude**: ARC-AGI-2에서 Gemini 3.1 Pro 77.1% 달성
- **OpenAI o 시리즈 (o3, o4)**: 수학 올림피아드 수준 문제에서 최강
- 이유: 두 회사 모두 "thinking" 모드에 대규모 수학 데이터 + 추론 강화 훈련

### 다국어 / 언어 번역

- **Gemini > GPT > Claude**: ⚠️ 추정 — Google이 처음부터 다국어 데이터를 균등하게 포함
- 구글 번역 서비스의 다국어 데이터가 간접 기여 가능성 있음

### 최신 정보 / 검색 연동

- **Gemini > GPT > Claude**: Google 검색 실시간 연동(Grounding)이 기본 내장
- GPT는 Bing 검색 연동, Claude는 Web Search 도구 별도

## 6. 결론 및 인사이트

1. **투명성 감소 추세**: GPT-3(2020)에서 가장 상세한 데이터 공개 → 이후 경쟁 심화로 모든 회사가 학습 데이터 비공개 기조로 전환

2. **합성 데이터가 핵심**: OpenAI를 필두로 "AI가 AI를 가르치는" 합성 데이터가 주력 학습 데이터로 부상. 인터넷 데이터 고갈 문제에 대한 해법

3. **코딩 왕좌**: Claude가 SWE-bench에서 1위지만, GPT-5.3-Codex의 터미널 특화, Gemini의 멀티모달 코딩 등 각 도메인별로 최적 모델이 다름

4. **멀티모달이 미래**: Google이 처음부터 멀티모달로 설계한 전략이 장기적으로 유리할 가능성 — 텍스트, 이미지, 오디오, 비디오를 통합한 데이터가 실세계 추론 능력의 핵심

5. **RLHF → CAI → RLAIF**: 학습 방법론의 진화 — 인간 라벨러 의존에서 AI 기반 피드백으로 전환하며 스케일 문제 해결
