#### 핵심 특징

| 항목 | 내용 |
|------|------|
| 언어 | Python |

| 운영 난이도 | 하~중 |
| **OpenClaw 궁합** | ⭐⭐⭐⭐⭐ |

#### 왜 OpenClaw와 궁합이 최고인가?

1. **MCP 서버 내장**: Scrapling이 MCP 서버를 제공하여 OpenClaw가 자연어 명령으로 Scrapling 기능 직접 호출 가능
   - LLMBase에 OpenClaw Skill로 등록됨 (`llmbase.ai/openclaw/scrapling-web-scraper`)
   - 토큰 사용량 절감: 대상 콘텐츠만 추출 후 AI에 전달
   
2. **StealthyFetcher**: Cloudflare Turnstile CAPTCHA 자동 우회
   - 브라우저 지문 스푸핑 내장
   - v0.4에서 101% 성능 향상
   
3. **적응형 파싱**: 웹사이트 구조 변경 시 자동으로 요소 재탐지
   - CSS 셀렉터가 깨져도 컨텍스트 기반으로 요소 재위치 추적

#### 실제 사용 시나리오

```python
from scrapling import StealthyFetcher

fetcher = StealthyFetcher()
page = fetcher.fetch("https://cloudflare-protected-site.com")
# Turnstile 자동 우회, 정상 HTML 반환
data = page.css("div.product-price::text")
```

#### 주의사항

보안 연구자들이 "24/7 실행 AI 에이전트 + Cloudflare 우회 프레임워크 조합"에 대한 윤리적 우려 제기. 실제 스크래핑 시 대상 사이트의 ToS 준수 필요.

| 운영 난이도 | 중 |

#### 왜 다른가?

기존 stealth 도구들은 JavaScript 레이어에서 지문을 조작하지만 Camoufox는 **Firefox 엔진의 C++ 레벨에서 지문을 가로챔**. JavaScript로는 탐지 불가능.

- **CreepJS 테스트**: 0% 탐지율
- **BrowserScan**: 0% 탐지율
- 기존 Python stealth 라이브러리 대비 압도적 우위

#### research-02 대비 위치

research-02에서 상위권이었던 `Crawlee 내장 지문 로테이션`보다 Cloudflare 우회 성공률이 높음. 단, Firefox 기반이라 Chromium 전제인 OpenClaw 스택과는 추가 연동 작업 필요.

### 1-3. Nodriver

**GitHub**: `ultrafunkamsterdam/nodriver` | Python

#### 핵심 특징

| 항목 | 내용 |
|------|------|
| 기반 | Chrome DevTools Protocol (CDP) 직접 연결 |
| 동적 페이지 | ✅ |
| anti-bot | ⭐⭐⭐⭐ (WebDriver 흔적 없음) |
| 유지보수성 | ⭐⭐⭐ (활발하지만 작은 팀) |
| 운영 난이도 | 중 |
| **OpenClaw 궁합** | ⭐⭐⭐ |

#### 왜 주목받나?

`undetected-chromedriver` 개발자가 만든 후계 프로젝트. WebDriver 프로토콜 자체를 사용하지 않아 WebDriver 탐지 방법이 원천 무효화됨.

- WebDriver 패치 불필요 → 패치 추적전(patch war) 탈출
- CDP로 직접 Chrome과 통신
- OpenClaw와 아키텍처가 유사 (둘 다 CDP 기반)

#### research-02의 `SeleniumBase UC Mode`와 비교

SeleniumBase UC Mode는 2025년 2월 유지보수 중단 → Nodriver가 실질적 후계자.

#### 두드러지는 기능

- **SmartCrawler**: 자연어로 "이 페이지에서 가격과 제품명 가져와" → 자동 추출

MCP 서버(`scrapegraph-mcp`)가 있어 OpenClaw 에이전트가 자연어로 스크래핑 지시 가능. LLM 네이티브 방식이므로 OpenClaw의 AI 파이프라인과 자연스럽게 결합.

## 5. 결론

1. **Scrapling MCP 서버**: OpenClaw와 직접 통합 가능한 anti-bot 도구 등장. 기존에는 Crawlee + 프록시 조합이 최선이었으나, 이제 OpenClaw 에이전트가 자연어로 Scrapling에 스크래핑 지시 가능.
