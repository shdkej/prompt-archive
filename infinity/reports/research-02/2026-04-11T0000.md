## 2. 브라우저 자동화 도구 비교

### Playwright (Microsoft) ⭐ 2026년 기준 최우선 추천

| 항목 | 내용 |
|------|------|
| 지원 브라우저 | Chromium, Firefox, WebKit |
| 언어 | Python, TypeScript/JS, Java, C# |
| JS 렌더링 | ✅ |
| anti-bot 내장 | △ (기본 수준, stealth 플러그인 필요) |
| 운영 난이도 | 중 |
| OpenClaw 궁합 | ⭐⭐⭐⭐⭐ (내부 CDP 엔진으로 Playwright 사용) |

**핵심**: OpenClaw Browser가 Playwright를 내부 CDP 엔진으로 사용. OpenClaw + Playwright는 가장 자연스러운 조합. auto-waiting, proxy-per-context, 멀티 브라우저 지원.

### Puppeteer (Google)

| 항목 | 내용 |
|------|------|
| 지원 브라우저 | Chrome/Chromium만 |
| 언어 | Node.js (TypeScript/JS) |
| JS 렌더링 | ✅ |
| anti-bot 내장 | △ (puppeteer-extra-plugin-stealth 필요) |
| 운영 난이도 | 중 |
| OpenClaw 궁합 | ⭐⭐⭐ (기존 Node.js 투자 있으면 선택 가능) |

**핵심**: Chrome 전용. 이미 Node.js 생태계에 투자가 있으면 선택지. 신규 프로젝트는 Playwright가 우위.

### Selenium

| 항목 | 내용 |
|------|------|
| 지원 브라우저 | Chrome, Firefox, Safari, Edge |
| 언어 | Python, Java, JS, C#, Ruby |
| JS 렌더링 | ✅ |
| anti-bot 내장 | △ (SeleniumBase UC Mode - Cloudflare 우회 지원) |
| 운영 난이도 | 중-상 |
| OpenClaw 궁합 | ⭐⭐ |

**핵심**: 레거시 선택지. SeleniumBase UC Mode는 Cloudflare Turnstile 우회에 강점이 있으나, 신규 프로젝트에서는 Playwright 우선.

## 5. Anti-Bot 대응 현황 (2026)

Cloudflare Bot Management가 확인하는 것:
- TLS 지문 (JA3/JA4)
- JavaScript 챌린지
- 브라우저 행동 패턴
- Turnstile CAPTCHA
- IP 평판

### 대응 전략별 효과

| 전략 | 효과 | 비용 |
|------|------|------|
| 데이터센터 프록시 | ❌ (쉽게 탐지됨) | 저 |
| 주거용(Residential) 프록시 | ✅ | 중-고 |
| Crawlee 내장 지문 로테이션 | ✅ (중간 수준 사이트) | 무료 |
| Nodriver / SeleniumBase UC Mode | ✅ (Cloudflare 특화) | 무료 |
| 관리형 서비스 (Bright Data, Scrapfly) | ⭐⭐⭐⭐⭐ | 고 |

> **주의**: 오픈소스 stealth 도구들은 Cloudflare의 지속적인 업데이트로 인해 효과가 떨어지는 추세. 강보호 사이트는 관리형 서비스 또는 residential 프록시 조합 필수.

## 6. 상황별 추천

### Case 1: OpenClaw에서 일반 동적 페이지 수집
**추천: Playwright (직접 연동)**
- OpenClaw가 내부적으로 Playwright 기반이므로 별도 도구 없이 Playwright 코드 활용
- 학습 비용 최소, 유지보수 단순

### Case 2: OpenClaw에서 LLM 파이프라인으로 바로 넣기
**추천: Firecrawl + OpenClaw 공식 연동**
- Markdown/JSON 출력으로 LLM 프롬프트에 바로 삽입 가능
- 공식 연동 문서 존재 (docs.firecrawl.dev)
- 강보호 사이트면 Scrapfly를 대안으로 고려

### Case 3: 대규모 정적 페이지 수집
**추천: Scrapy (+ scrapy-playwright)**
- 수천 페이지/분 처리 가능
- 동적 페이지는 scrapy-playwright 플러그인으로 보완

### Case 4: anti-bot이 강한 사이트 (Cloudflare, LinkedIn 등)
**추천: Crawlee + Residential 프록시** 또는 **관리형 서비스(Scrapfly, Bright Data)**
- Crawlee: 오픈소스 중 최강 stealth (브라우저 지문 로테이션, 세션 관리)
- Scrapfly: 98% 성공률 주장, Firecrawl 대비 3배 이상 성공률

### Case 5: 빠른 MVP / 코딩 없이 수집
**추천: Octoparse, ParseHub (노코드)**
- 셀럽, 비기술 직군용
- OpenClaw 연동은 별도 API 작업 필요

### Case 6: Python 생태계에서 anti-bot까지 자체 처리
**추천: Crawlee (Python v1.0)**
- 2025.09 기준 Python 버전 안정화, GitHub 6,000+ stars
- 프록시/지문/세션 관리 기본 내장

## 8. 결론 및 실행 제안

OpenClaw 환경에서 **데이터를 계속 가져오는** 상황이라면:

1. **기본 수집**: Playwright 직접 활용 (OpenClaw 기본 내장)
2. **LLM 연동 필요**: Firecrawl 추가 (공식 연동 지원)
3. **보호된 사이트**: Crawlee + Residential 프록시로 전환
4. **대규모 스케일**: Scrapy + scrapy-playwright 조합

단계적 접근으로 **불필요한 도구 도입 없이** 상황에 따라 확장하는 전략이 유지보수성과 운영 난이도 관점에서 최적.
