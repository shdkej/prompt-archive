## remaining_risks

1. **하이드레이션 시 날짜 텍스트 잠깐 빈 자리** — `useEffect` 패턴이라 첫 페인트 ~수 ms 동안 한 칸 공백. `suppressHydrationWarning` + `" "` placeholder로 레이아웃 점프는 방지했지만, 시각적으로 깜빡임이 거슬릴 경우 server-render 가능한 ISO 기반 표시(timezone 무관 포맷)로 전환을 고려.
2. **기존 사용자의 시드 데이터** — 이미 앱을 한 번 열어 본 사용자의 localStorage엔 `MOCK_DEEDS`가 저장되어 있음. 본 패치는 신규 사용자만 빈 상태로 시작. 기존 사용자는 `/me` → "기록 초기화"로 자발 정리하거나 추후 마이그레이션 추가 필요. (운영자 결정 필요 — 자동 강제 클리어는 본 사이클 범위 밖.)
3. **`INITIAL_VIRTUE = 612`는 그대로** — 사전 리포트도 이 부분은 분리 의제로 둠. 빈 deeds + 612덕이면 시작 단계가 즉시 "고슴도치"인 점은 게임 디자인 결정사항. 진정한 0부터 시작하려면 `INITIAL_VIRTUE = 0`도 함께 고려해야 하나, 본 사이클은 사전 리포트의 명시 항목만 처리.
4. **ESLint/`next lint` 사전 실패** — 본 패치와 무관한 Next 16 업그레이드 잔여 이슈. 별도 PR로 `package.json`의 lint 스크립트를 `eslint .` 또는 `eslint src` 로 갱신 + flat config 재정렬 필요. 본 사이클 verification은 `tsc --noEmit` + `next build`로 갈음.
5. **`/me`의 데이터 내보내기 카피 "(아직 mock)"** — 본 사이클 범위 밖. 실제 내보내기는 동작하므로 카피만 후속 정리 대상.
6. **`src/lib/format.ts`의 고정 NOW(`2026-05-13T18:00:00+09:00`)** — `formatTimeAgo`/`formatRelativeDay`의 기준 시각이 여전히 mock 날짜. 신규 사용자가 오늘 deed를 추가하면 `formatTimeAgo`는 mock NOW 기준으로 계산되어 부정확한 "어제/N분 전"이 표시될 수 있음. 사전 리포트에 미포함된 별개 이슈 — 후속 사이클에서 분리 처리 권장.
