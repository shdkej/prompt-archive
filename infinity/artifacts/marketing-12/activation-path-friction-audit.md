# Virtue 활성화 경로 마찰 감사표

> prelaunch 단계 관찰 전용. 성패 판정 금지.
> cloud draft — 최종본은 virtue-rebirth-app/docs/activation-path-friction-audit.md (canonical)

## 0. 전제

- 이 문서는 Reforge/Appcues 결과 기반 온보딩 관점에서 `/` → `/add` → 채점 → 저장 → 누적 피드백 경로의 마찰을 J1-J4 잡별로 정리한다.
- "결과 기반 온보딩": 신규 사용자는 기능 설명이 아니라 첫 가치 결과(aha moment)에 빨리 닿아야 한다. 마찰이 그 경로를 막으면 "나쁜 마찰", 가치 결과를 향한 집중을 돕거나 출력 품질을 높이면 "좋은 마찰"이다.
- 기존 이벤트(`add_flow_started`, `deed_judged`, `deed_saved`, `level_up_viewed`)만 사용. 신규 이벤트/속성/코드/카피/대시보드 변경 0.
- `activation-milestone-ladder.md`(setup→aha→habit), `first-session-jtbd-matrix.md`(J1~J4 정의), `first-real-user-baseline-template.md`(첫 10명 관찰 양식)를 계승하고 재정의하지 않는다.
- prelaunch 단계이므로 어떤 마찰 지표도 "성공/실패" 판정 금지. 관찰·기록 전용.

## 1. 경로 단계 정의

| 단계 | 화면/액션 | 진입 이벤트 | 의미 |
|---|---|---|---|
| **A. 홈** | `/` (랜딩) | — | 첫 인상, 가치 약속 인식 |
| **B. Add 진입** | `/add` (덕행 추가 시작) | `add_flow_started` | 첫 행동 의도 — setup moment |
| **C. 채점** | AI 판정 요청·결과 확인 | `deed_judged` | J3 aha moment; J1/J2/J4에서는 선택적 경유 |
| **D. 저장** | 덕행 저장 완료 | `deed_saved` | J1/J2/J4 aha moment |
| **E. 누적 피드백** | 레벨업 화면 노출 | `level_up_viewed` | J2 habit moment 전조; 누적 가시화 |

> J3는 C(채점)가 aha moment이므로 D(저장)는 선택적. J1/J2/J4는 D(저장)가 aha moment이며 C는 경유하거나 건너뜀.

## 2. 마찰 유형 기준

| 유형 | 정의 | 판별 기준 |
|---|---|---|
| ✅ **좋은 마찰** | 첫 가치 결과 도달을 돕거나 출력 품질을 높이는 의도적 멈춤 | 제거 시 사용자가 가치를 못 느끼거나 품질이 떨어짐 |
| ❌ **나쁜 마찰** | 첫 가치 결과 도달을 막거나 지연시키는 불필요한 단계/모호함 | 제거 시 경로가 빨라지고 이탈이 줄 것으로 예상됨 |
| 🔲 **보류** | 잡에 따라 부호가 다르거나, 실 사용자 관찰 전에 판정 불가 | 첫 10명 관찰 후 재분류 |

## 3. J1-J4별 경로 마찰 감사표

### J1 기록형 — 오늘 한 일을 빠르게 덕행으로 남기고 싶다

| 경로 단계 | 마찰 항목 | 유형 | 근거 및 관찰 포인트 |
|---|---|---|---|
| **A. 홈** | 앱 목적 설명 없이 대시보드 바로 노출 | 🔲 보류 | 기존 사용자에겐 빠름(✅), 첫 방문자엔 방향 부재(❌) → 첫 10명 관찰 필요 |
| **A. 홈** | "덕행 추가" CTA 위치와 시인성 | 🔲 보류 | 빈 상태에서 CTA가 충분히 눈에 띄는지 미검증 |
| **B. Add 진입** | 덕행 설명 직접 타이핑 필요 | ✅ 좋은 마찰 | 자기 반성 유도, 출력 품질 향상. 제거 시 빈 덕행 저장 위험 |
| **B. Add 진입** | AI 판정 옵션 노출 (J1은 원하지 않을 수 있음) | 🔲 보류 | 강제 경유면 ❌. 선택 가능이면 ✅. 코드 확인으로 판별 가능 |
| **C. 채점** | J1이 채점을 건너뛸 수 있는가 | 🔲 보류 | 채점 필수 여부 `src/app/add/page.tsx` 확인 필요. 건너뛸 수 있으면 J1 경로 단축 ✅ |
| **D. 저장** | 저장 완료 후 피드백 (성공 메시지) | ✅ 좋은 마찰 | 가치 결과 확인 순간. 없으면 aha moment 미발화 → ❌ |
| **D. 저장** | 저장 후 다음 행동 유도 없음 | ❌ 나쁜 마찰 | J1이 저장 후 무엇을 해야 할지 경로 없으면 이탈 위험 |
| **E. 누적 피드백** | J1은 레벨업 피드백 필요성 낮음 | 🔲 보류 | J1의 aha는 저장 자체. 레벨업은 보너스. 첫 10명 반응 관찰 |

### J2 누적형 — 기록이 쌓이는 걸 보면서 성장감을 느끼고 싶다

| 경로 단계 | 마찰 항목 | 유형 | 근거 및 관찰 포인트 |
|---|---|---|---|
| **A. 홈** | 첫 방문 시 누적 0 상태 — 빈 대시보드 | ❌ 나쁜 마찰 | 누적감이 목적인 J2에게 빈 화면은 가치 약속 미전달 |
| **A. 홈** | 빈 상태 CTA와 카피 품질 | 🔲 보류 | `empty-state-first-action-audit.md` 참조. 이 감사표는 상태만 기록 |
| **B. Add 진입** | 첫 덕행 입력 시 예시/힌트 부재 | ❌ 나쁜 마찰 | 누적을 원하지만 첫 항목 선택이 막히면 이탈. "뭘 써야 할지 모름" |
| **C. 채점** | 채점 결과가 누적 카운터에 영향 없음 | ✅ 좋은 마찰 | J2는 저장이 누적 단위. 채점은 선택 경유해도 목표에 지장 없음 |
| **D. 저장** | 저장 직후 누적 수 변화 피드백 (카운터 증가 표시) | ✅ 좋은 마찰 | 누적의 첫 가시화. 없으면 J2 aha moment 미발화 위험 |
| **E. 누적 피드백** | 레벨업 타이밍 (너무 빠르거나 너무 늦음) | 🔲 보류 | 레벨업이 J2의 habit moment 전조. 타이밍이 J2 기대와 맞는지 첫 10명 관찰 |
| **E. 누적 피드백** | 레벨업 이후 다음 목표 가시화 부재 | ❌ 나쁜 마찰 | 레벨업 후 "다음 레벨까지 X개" 없으면 J2 momentum 단절 위험 |

### J3 AI 호기심형 — AI가 내 덕행을 어떻게 판정하는지 보고 싶다

| 경로 단계 | 마찰 항목 | 유형 | 근거 및 관찰 포인트 |
|---|---|---|---|
| **A. 홈** | AI 판정 기능 노출 여부 | 🔲 보류 | 홈에서 AI 기능을 암시하지 않으면 J3 유입 자체가 줄어들 수 있음 |
| **B. Add 진입** | 덕행 입력 → AI 판정 요청까지의 단계 수 | ❌ 나쁜 마찰 (잠재) | J3의 목적은 AI 판정 경험. 단계 많으면 이탈 위험. 코드 확인 필요 |
| **B. Add 진입** | 덕행 설명 입력 강제 (판정 전) | ✅ 좋은 마찰 | 빈 입력으로 AI 판정 요청 불가 → 품질 있는 판정 유도 |
| **C. 채점** | AI 판정 대기 시간 | 🔲 보류 | 체감 대기가 길면 ❌. 즉각적이면 ✅. 실 사용자 측정 필요 |
| **C. 채점** | AI 판정 결과의 가독성과 설명 품질 | ✅ 좋은 마찰 | 결과 설명이 J3의 aha moment. 판정 근거 없으면 호기심 미충족 → ❌ |
| **C. 채점** | 재판정 옵션 (`deed_rerolled`) | ✅ 좋은 마찰 | 호기심 충족 레이어 추가. 남용 방지 캡(`deed_save_capped` 코드) 확인됨 |
| **D. 저장** | 판정 후 저장 강제 여부 | 🔲 보류 | J3는 저장보다 판정이 목적. 강제면 ❌. 선택이면 ✅ |
| **E. 누적 피드백** | J3는 레벨업보다 판정 횟수 누적 관심 | 🔲 보류 | 판정 횟수 가시화 현재 미구현. 관찰 후 재분류 |

### J4 회고형 — 시간이 지난 뒤 내 덕행 기록을 돌아보고 싶다

| 경로 단계 | 마찰 항목 | 유형 | 근거 및 관찰 포인트 |
|---|---|---|---|
| **A. 홈** | 과거 기록 접근성 (히스토리 진입점) | ❌ 나쁜 마찰 (잠재) | J4의 두 번째 방문 목적은 기록 회고. 히스토리가 묻혀있으면 이탈 |
| **A. 홈** | 최근 덕행 요약 노출 | ✅ 좋은 마찰 | 회고 욕구를 홈에서 자극. 없으면 J4 재방문 동기 약화 |
| **B. Add 진입** | 새 덕행 추가 경로 (회고 후 행동으로 이어지는가) | 🔲 보류 | J4는 회고 후 새 기록으로 이어질 수 있음. 경로 연결 여부 관찰 필요 |
| **C. 채점** | J4의 채점 필요성 | 🔲 보류 | 회고가 목적. 채점이 기록 품질 향상에 도움(✅) 또는 번거로움(❌) — 잡별 부호 불명 |
| **D. 저장** | 저장 시 날짜/시간 가시화 | ✅ 좋은 마찰 | 회고의 시간 축 형성. 없으면 J4 가치 약속 미전달 |
| **D. 저장** | 저장 후 과거 기록 비교 유도 없음 | ❌ 나쁜 마찰 | J4는 "오늘 것과 어제 것 비교"를 원함. 연결 경로 없으면 회고 루프 단절 |
| **E. 누적 피드백** | 레벨업 화면에서 과거 기록 요약 노출 여부 | 🔲 보류 | 과거 기록 요약 있으면 J4에 ✅. 현재 미확인 → 코드/UI 확인 필요 |

| 홈 CTA 시인성 | 첫 방문 사용자가 추가 버튼을 즉시 찾는가 (수기 관찰) | `first-real-user-baseline-template.md` |
| AI 판정 옵션 강제/선택 | `src/app/add/page.tsx` 코드 확인으로 즉시 판별 | `deed_judged` 발화 조건 라인 확인 |
| 저장 후 다음 행동 유도 | J별 재방문율 + "다음 방문 이유" 정성 질문 | `time-to-value-observation-brief.md` second value |
| 레벨업 타이밍 | 첫 레벨업까지 `deed_saved` 횟수 수기 관찰 | `activation-milestone-ladder.md` habit moment |
