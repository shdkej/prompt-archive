# Virtue 경쟁 대안 기반 포지셔닝 브리프

> prelaunch 단계 내부 기획 문서. 포지셔닝/copy 논의의 입력값이며, 배포 카피·트래킹·코드 변경 지시서가 아니다.
> cloud draft — 최종본은 virtue-rebirth-app/docs/competitive-alternatives-positioning-brief.md

## 0. 배경 및 전제

- Virtue는 prelaunch 단계. 작은 지표를 성패로 판정하지 않는다.
- "경쟁 대안"은 경쟁사가 아니라 **사용자가 지금 Virtue 대신 하고 있는 행동**이다 (JTBD 전환 개념).
- 이 브리프는 J1-J4별로 경쟁 대안, 차별 속성, 고객 가치, 현재 첫 화면 신호, 검증 질문을 고정해 포지셔닝·copy 논의를 근거 있게 만든다.
- 기존 이벤트(`add_flow_started`, `deed_judged`, `deed_saved`, `level_up_viewed`)만 인용. 신규 이벤트·속성·코드·카피·대시보드 변경 0.
- `first-session-jtbd-matrix.md`, `pmf-response-analysis-rubric.md`, `minimum-viable-audience-brief.md`, `copy-spec.md` 충돌 0 원칙.

## 1. J1~J4 경쟁 대안 포지셔닝 표

### J1 기록형 — 오늘 한 일을 빠르게 덕행으로 남기고 싶다

| 항목 | 내용 |
|---|---|
| **경쟁 대안** | ① 아무것도 안 함(기록하지 않음) ② 메모장/노트앱(Samsung Notes, 카카오톡 메모) ③ 일기 앱(Day One, Notion) |
| **Virtue 차별 속성** | "덕행"이라는 의미 프레임으로 기록 고정 → 메모가 아닌 인격 행동의 영구화. `deed_saved` 즉시 피드백으로 기록 완료 신호 제공 |
| **고객 가치** | "오늘도 좋은 일 했다"는 자기 인정이 기록 행동 하나로 완결됨. 별도 앱 스위칭 없이 덕행 카테고리로만 필터된 나의 기록 |
| **현재 첫 화면 신호** | "덕력"·"환생종" 신호는 J1에게 기록 편의성 약속보다 세계관 진입으로 읽힐 수 있음. "빠르게 기록" 신호 강도 🔲 보류 |
| **첫 10명 검증 질문** | "오늘 잘 한 일을 지금 어디에 기록하나요?" / "Virtue 전에 좋은 행동을 기록하던 앱이나 방법이 있었나요?" |

### J2 누적형 — 기록이 쌓이는 걸 보면서 성장감을 느끼고 싶다

| 항목 | 내용 |
|---|---|
| **경쟁 대안** | ① 습관 추적 앱(Habitica, Loop, Streaks) ② 피트니스/러닝 앱의 누적 통계 ③ 게임의 경험치·레벨업 시스템 |
| **Virtue 차별 속성** | 수치가 아닌 "덕행"이라는 인격적 차원의 누적. 레벨업(`level_up_viewed`)이 게임 포인트가 아닌 실제 행동 패턴의 가시화 |
| **고객 가치** | "습관 앱의 체크는 의미 없었는데 Virtue는 내 행동의 질이 쌓인다는 느낌". 환생종 변화로 누적의 정체성 전환 시각화 |
| **현재 첫 화면 신호** | "환생종" 레벨 개념이 J2에게 강한 누적 신호 ✅. 빈 첫 화면(0 상태)에서 누적 가능성 암시 강도는 🔲 보류 |
| **첫 10명 검증 질문** | "지금 사용하는 습관 추적 앱이 있나요? 거기서 가장 뿌듯한 순간은 언제인가요?" / "Virtue에서 뭔가 쌓인다는 느낌이 언제 처음 왔나요?" |

### J3 AI 호기심형 — AI가 내 덕행을 어떻게 판정하는지 보고 싶다

| 항목 | 내용 |
|---|---|
| **경쟁 대안** | ① ChatGPT에 "이게 착한 일인가요?" 직접 질문 ② 윤리·철학 커뮤니티(Reddit r/philosophy, 온라인 카페) ③ 불교·유교 관련 자기계발 콘텐츠 |
| **Virtue 차별 속성** | 일반 대화 AI가 아닌 "내 행동"에 즉각 특화된 판정 컨텍스트. `deed_judged` + `deed_rerolled`로 판정 재요청·비교가 가능한 게임형 구조 |
| **고객 가치** | "ChatGPT는 너무 일반적인데, Virtue는 내 구체적 행동에 맞춘 판정을 즉시 준다". 판정 결과가 저장과 연결되어 컬렉션이 됨 |
| **현재 첫 화면 신호** | 첫 화면에서 AI 판정 기능이 명시적으로 암시되는지 확인 필요 🔲 보류. J3 유입은 이 신호에 민감 |
| **첫 10명 검증 질문** | "내가 한 일이 좋은 일인지 AI에게 물어본 적 있나요? 어디서요?" / "Virtue의 AI 판정 결과를 보고 어떤 느낌이었나요?" |

### J4 회고형 — 시간이 지난 뒤 내 덕행 기록을 돌아보고 싶다

| 항목 | 내용 |
|---|---|
| **경쟁 대안** | ① 일기 앱(Day One, Notion 회고 페이지) ② 연말/주간 회고 의식(물리 노트, Google Docs 회고) ③ SNS 타임라인으로 과거 행동 추적 |
| **Virtue 차별 속성** | 일기의 잡다한 내용 없이 "덕행"이라는 단일 필터로 정리된 회고. 날짜별 `deed_saved` 기록이 인격적 행동 패턴의 타임라인이 됨 |
| **고객 가치** | "Notion 회고는 쓰는 데 1시간 걸리는데 Virtue는 쌓인 게 바로 내 기록이다". 회고를 따로 쓸 필요 없이 매일 저장된 덕행이 자동으로 회고 소재가 됨 |
| **현재 첫 화면 신호** | 최근 덕행 요약/히스토리 진입점의 첫 화면 가시성 🔲 보류 (J4 재방문 동기와 직결) |
| **첫 10명 검증 질문** | "한 달 전의 나와 지금을 비교해보고 싶을 때 어떻게 하나요?" / "Virtue의 기록을 다시 열어본 적 있나요? 그때 어떤 느낌?" |

## 2. 이기는 가치(차별 속성) 요약

| 잡 | 경쟁 대안의 핵심 약점 | Virtue가 이기는 이유 |
|---|---|---|
| J1 기록형 | 일반 메모/일기는 "덕행" 카테고리가 없어 나중에 찾기 어려움 | 행동 기록이 즉각 덕행으로 분류·완결됨 |
| J2 누적형 | 습관 앱은 수치 기록이지 인격적 의미 없음 | 누적이 레벨·환생종이라는 인격 정체성 변화로 연결됨 |
| J3 AI 호기심형 | ChatGPT는 "내 행동"에 특화된 판정이 아닌 일반 답변 | 내 행동 텍스트에 즉각 판정+재판정 가능한 특화 컨텍스트 |
| J4 회고형 | 일기/회고는 별도 시간·노력을 요구 | 매일 저장된 덕행이 자동으로 회고 소재가 됨 — 회고 부담 0 |

## 3. 현재 첫 화면 신호 요약

| 잡 | 현재 신호 강도 | 확인 필요 사항 |
|---|---|---|
| J1 기록형 | 🔲 약함·보류 | "빠른 기록" 메시지가 홈에 노출되는가 |
| J2 누적형 | ✅ 강함 | "환생종" 레벨업이 누적 신호 전달. 빈 상태 신호는 🔲 보류 |
| J3 AI 호기심형 | 🔲 보류 | AI 판정 기능이 홈 첫인상에 명시되는가 |
| J4 회고형 | 🔲 약함·보류 | 히스토리/최근 덕행 요약의 첫 화면 진입점 가시성 |

## 4. 첫 10명 검증 계획

> 관찰 전용. 성패 판정·전환율 임계값 설정 금지 (prelaunch 경계).

| 검증 목표 | 방법 | 경쟁 대안 확인 포인트 |
|---|---|---|
| 어떤 대안에서 왔는가 | 첫 대화·DM 질문 1개 | "Virtue 전에 덕행/좋은 행동 기록에 뭘 쓰셨나요?" |
| 어떤 신호로 앱을 이해했는가 | 첫 세션 수기 관찰 | "처음 열었을 때 이 앱이 뭐하는 건지 어떻게 이해하셨나요?" |
| 잡 분류 | 첫 세션 행동 관찰(`deed_saved` vs `deed_judged` 우선) | J3이면 채점 먼저, J1/J2/J4이면 저장 먼저 |
| 이기는 가치 검증 | 잡별 핵심 검증 질문(§1 참조) | 각 잡의 "고객 가치" 가설이 실제로 체감되는가 |
