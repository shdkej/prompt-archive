# wiki-04: agent-wiki 전체 문서 자동 네비게이션

> Intent: wiki-04 | 작성: 2026-04-24T09:00 | 업데이트: 2026-04-25T09:00 (JS 변환) | L2 승인 대기 (agent-wiki push)

## 문제 분석

### 현재 상황
- wiki-02: Docsify + GitHub Pages 구축 (`loadSidebar: true`)
- wiki-03: 모바일 토글/오버레이/자동닫힘 UX 구현
- `_sidebar.md`: 수동으로 유지되는 하드코딩 메뉴
  - 새 문서 추가 시 수동으로 `_sidebar.md`도 업데이트해야 함
  - 누락된 문서가 모바일에서 탐색 불가

### 요구사항
- 전체 문서 구조를 자동으로 반영하는 사이드바
- 모바일에서도 모든 문서 탐색 가능
- 새 문서 추가 시 수동 작업 없이 자동 업데이트
- **스크립트 언어: JavaScript (Node.js)** ← wiki-05에서 변경

## 솔루션 선택: GitHub Actions + 자동 생성 스크립트

### 비교 검토

| 방식 | 장점 | 단점 | 선택 |
|------|------|------|------|
| A. docsify-auto-sidebar 플러그인 | 클라이언트 사이드, 설정 간단 | GitHub API rate limit, 파일 구조 노출 제한 | 보류 |
| **B. GitHub Actions 자동 생성** | 항상 정확, 빌드 타임 처리, 속도 빠름 | 워크플로 설정 필요 | **채택** |
| C. 수동 유지 | 제어 가능 | 현재 문제의 원인 | 기각 |

**채택 근거**: Docsify가 이미 `loadSidebar: true`로 `_sidebar.md`를 사용 중.
GitHub Actions으로 `_sidebar.md`를 자동 생성하면 기존 설정 변경 없이 해결 가능.
Node.js는 GitHub Actions ubuntu-latest에 기본 내장.

## 구현 파일

### 1. `.github/workflows/auto-sidebar.yml`

```yaml
name: Auto-generate _sidebar.md

on:
  push:
    branches: [main]
    paths:
      - '**/*.md'
      - '!_sidebar.md'
      - 'scripts/generate_sidebar.js'
  workflow_dispatch:

permissions:
  contents: write

jobs:
  generate-sidebar:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Generate _sidebar.md
        run: node scripts/generate_sidebar.js > _sidebar.md

      - name: Commit and push if changed
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add _sidebar.md
          git diff --staged --quiet || (
            git commit -m "chore: auto-update _sidebar.md [skip ci]"
            git push
          )
```

### 2. `scripts/generate_sidebar.js`

```javascript
#!/usr/bin/env node
// agent-wiki _sidebar.md 자동 생성기

const fs = require('fs');
const path = require('path');

const EXCLUDE_FILES = new Set(['_sidebar.md', '_navbar.md', '.nojekyll']);
const EXCLUDE_DIRS = new Set(['.git', '.github', 'scripts', 'node_modules']);

function getTitle(filepath) {
  try {
    const content = fs.readFileSync(filepath, 'utf-8');
    for (const line of content.split('\n')) {
      const trimmed = line.trim();
      if (trimmed.startsWith('# ')) {
        return trimmed.slice(2).trim();
      }
    }
  } catch (e) {}
  return path.basename(filepath, path.extname(filepath));
}

function scanDir(dir, base, indent = 0) {
  const lines = [];
  let entries;
  try {
    entries = fs.readdirSync(dir, { withFileTypes: true });
  } catch (e) {
    return lines;
  }

  entries.sort((a, b) => {
    if (a.isDirectory() !== b.isDirectory()) return a.isDirectory() ? -1 : 1;
    return a.name.toLowerCase().localeCompare(b.name.toLowerCase());
  });

  for (const entry of entries) {
    if (entry.name.startsWith('.') || entry.name.startsWith('_')) continue;

    if (entry.isDirectory()) {
      if (EXCLUDE_DIRS.has(entry.name)) continue;
      const subLines = scanDir(path.join(dir, entry.name), base, indent + 1);
      if (subLines.length > 0) {
        lines.push(`${'  '.repeat(indent)}- **${entry.name}**`);
        lines.push(...subLines);
      }
    } else if (entry.isFile() && entry.name.endsWith('.md')) {
      if (EXCLUDE_FILES.has(entry.name)) continue;
      const filepath = path.join(dir, entry.name);
      const title = getTitle(filepath);
      const relPath = path.relative(base, filepath).replace(/\\/g, '/');

      if (entry.name === 'README.md') {
        const dirPath = path.relative(base, dir).replace(/\\/g, '/');
        if (dirPath === '.') continue;
        lines.unshift(`${'  '.repeat(indent)}- [${title}](${dirPath}/)`);
      } else {
        lines.push(`${'  '.repeat(indent)}- [${title}](${relPath})`);
      }
    }
  }
  return lines;
}

const base = process.cwd();
const output = [
  '<!-- Auto-generated by GitHub Actions — do not edit manually -->',
  '- [Home](/)',
  '',
  ...scanDir(base, base, 0),
];
console.log(output.join('\n'));
```

## 배포 절차 (L2 승인 후 실행)

```bash
# 1. agent-wiki 클론
git clone https://github.com/shdkej/agent-wiki.git /tmp/agent-wiki-wiki04
cd /tmp/agent-wiki-wiki04

# 2. 디렉토리 생성
mkdir -p .github/workflows scripts

# 3. 워크플로 파일 생성
cat > .github/workflows/auto-sidebar.yml << 'YAML_EOF'
(위 YAML 내용)
YAML_EOF

# 4. JS 스크립트 생성
cat > scripts/generate_sidebar.js << 'JS_EOF'
(위 JavaScript 내용)
JS_EOF

# 5. 로컬 테스트 (Node.js 필요)
node scripts/generate_sidebar.js

# 6. 커밋 & 푸시
git add .github/workflows/auto-sidebar.yml scripts/generate_sidebar.js
git commit -m "feat: auto-generate _sidebar.md via GitHub Actions (JS)"
git push origin main

# 7. GitHub Actions 실행 확인
# → 워크플로가 _sidebar.md를 자동 생성하고 커밋
# → GitHub Pages 재빌드 → https://shdkej.github.io/agent-wiki/ 확인
```

## 주의사항

- `[skip ci]` 태그로 자동 생성 커밋이 무한 루프 트리거 방지
- `paths: '!_sidebar.md'` 조건으로도 이중 방지
- `permissions: contents: write` 필요 (워크플로가 커밋하려면)
- 디렉토리가 비어있으면 섹션 헤더 미출력 (subLines 체크)
- Node.js는 GitHub Actions `ubuntu-latest`에 기본 탑재 (별도 설치 불필요)
