'우리 같이 생산자가 되어 봅시다' 라는 주제로 Threads에 글을 작성한다. 이 문서는 **작성 가이드 → 톤 레퍼런스 → 발행 전 검수**를 한곳에 담은 단일 원천이다. 작성과 검수 모두 이 문서를 기준으로 한다.

작성 전·검수 전 같은 레포 `WRITING_TECHNIQUES.md`(퇴고 체크리스트)와 `TASTE.md` §2 「글 톤 & 회고 패턴」(Threads 문체 섹션)을 함께 참조한다. 특히 골격 회전 규칙(승인 샘플의 톤만 참고, 문장 골격 재사용 금지)을 지킨다.

## 1. 글 작성 가이드

### STEP 0: 각도 선언 (쓰기 전 필수)

사건형 / 판단형 / 반성형 / 관찰형 / 대화형 / 숫자형 중 **하나를 먼저 고르고** 그 각도로만 쓴다.

- "A했는데 정작 B" 자기 아이러니 고백은 여러 각도 중 하나일 뿐 **기본값이 아니다**. 시드에 아이러니가 있어도 관찰형·숫자형·대화형으로 풀 수 있는지 먼저 본다. 숫자 오프닝·시점 선언("7월 1일, ~하기로 했습니다")도 마찬가지로 하나의 각도일 뿐이다.
- 쓰기 전에 **최근 7개 글(같은 세션 후보 포함)의 첫 줄·마무리 구조를 확인**하고, 각도·첫 줄 골격·마무리 장치가 겹치면 폐기하고 다른 각도로 다시 쓴다.

### 첫 줄 = 선언

첫 줄은 요약이 아니라 **선언**이다. 주장 / 반전 / 선택 압력 / 구체 대비 중 하나로 독자를 세운다.

- 금지: 질문 오프닝(`~뭔지 앎?`류 포함), 작성자 사정 요약("퇴사 전 마지막 작업은 ~입니다"), 날짜+결정 보고("7월 1일, ~하기로 했습니다"), 추상 요약, **격언·잠언형 일반 명제**("동시에 다섯 개를 붙잡으면, 한 개도 제대로 못 붙잡는다" — 단독으로 떼어도 성립하는 문장은 첫 줄이 될 수 없다. 승인 샘플 "여기서 대부분의 초보 창업가가 실패한다"는 '여기서'로 상황에 못 박혀 있어서 허용).
- 글에서 가장 센 문장(반전·인용·숫자 대비)이 중반에 있으면 **그 문장을 첫 줄로 끌어올린다.**
- 오프닝 각도 예 — **각도만 참고하고 문장 골격은 복제 금지**:
  - 관찰 선언: "여기서 대부분의 초보 창업가가 실패한다."
  - 숫자 대비: "네이버는 신혼여행을 밀어줬고, 유튜브는 세계일주를 밀어줬다."
  - 행동/결정 선언: 내가 내린 결정을 숫자와 함께.
  - 발화 인용: 실제 들은 말·한 말을 따옴표 그대로.

### 계정 테마 (2026-07-18)

Threads와 Instagram의 반복 테마는 **미니멀, 실용성, 소비취향 공유**다.

- 여행 글도 장소 감상만 쓰지 않는다. `무엇을 샀나 / 무엇을 안 샀나 / 무엇을 남겼나 / 어떤 선택이 편했나` 중 하나가 보여야 한다.
- 첫 줄은 감상 요약보다 취향이 드러나는 선택 문장으로 시작한다. 예: `쾰른에서 산 건 거의 없고, 남긴 건 결혼사진이었습니다.`
- 소비취향은 과시가 아니라 기준 공유다. 가격·브랜드·물건이 없어도 `시간`, `동선`, `사진`, `공간`, `짐`, `기록 방식`이 소비 선택이면 소재가 된다.
- 미니멀은 휑한 문장이 아니라 덜어낸 기준이다. "좋았다"보다 "왜 이 하나를 남겼는지"를 쓴다.
- 실용성은 체크리스트 남발이 아니라 다음 사람이 따라 할 수 있는 작은 조건 1개다. 예: 오전/낮 시간, 사람 적음, 일상복+웨딩복 콜라주, 숙소 근처 공원.

### 썸네일 / 첫 이미지

Threads에 사진이 붙거나 Instagram으로 재사용될 글은 본문 첫 줄과 썸네일 문구를 한 세트로 본다.

- 썸네일 문구는 8~16자 안팎으로 짧게, 본문 첫 줄을 그대로 반복하지 않는다.
- 좋은 썸네일은 장면을 설명하지 않고 선택을 보여준다. 예: `산 것보다 남긴 것`, `쾰른에서 남긴 장면`, `낮의 결혼사진`.
- 사진 구성은 1장 단독보다, 의미가 있으면 전날 일상복 장면 + 다음날 웨딩복 장면처럼 대비가 보이는 콜라주를 우선한다.
- 썸네일과 본문이 함께 같은 테마(미니멀/실용성/소비취향 중 하나)를 가리키지 않으면 다시 쓴다.

### 몸통

- 250자 내외, 3~5줄에 핵심. 2~3줄마다 빈 줄로 호흡.
- **독자 관점으로 쓴다**: "내가 뭘 했나" 나열이 아니라 읽는 사람이 얻는 것·공감할 것 중심. 첫 문장부터 독자를 세운다.
- 성과 수치는 반드시 시행착오·현재진행형과 병기한다. ("3명의 유료 사용자 확보 ㅎㅎ", "공부 할게 넘 많다" 톤 — 이력서식 나열 금지)
- 사실만 쓴다. 수치·도구명·날짜·장면 중 구체 디테일 최소 1개. **시드(소재)에 없는 사실·수치·디테일을 지어내지 않는다** — 디테일이 부족하면 지어내지 말고 사용자에게 물어본다.
- 경험, 감정, 유머, 비유, 호기심을 넣되 문장은 쉽게 쓴다.
- 길게 초안을 쓴 뒤 필요 없는 부분을 지운다.
- `적/의/것/성`, `하고 있다`, `할 수 있다`, `관한`, `대한`, `있는/있어서`를 퇴고 때 줄인다.

### 마무리

- **자기 구체 문장으로 닫는다.** 내 사건은 내 사건에서 끝낸다.
- 금지: 격언화(`결국 ~인 듯`, `~것 같다`, `~에 가깝다`, `~할 때가 있다`로 일반화하며 닫기), **독자 일반화 조언**("~하신다면 ~해보세요", "~한번 들여다보세요" — 자기 사건을 독자 교훈으로 확장하는 마감).
- 마무리 장치(독자 질문 / 다음 액션 선언 / 괄호 독백)는 **글당 최대 1개, 안 넣어도 된다.** 특히 `(...다..)` 괄호 두점 독백은 틱이 되기 쉬우니 예외적으로만 쓴다.
- 인용·질문을 넣을 땐 격언이 아니라 구체 문장으로.

### 금지 골격 (작성 후 자가 검사)

하나라도 걸리면 각도를 바꿔 다시 쓴다:

1. `주장 → 오늘 사례 → 일반화 교훈` 3단 고정
2. 첫 줄 `오늘은 X보다 Y` / `오늘 배운 것:` 오프닝
3. 첫 줄 "A했는데 정작 B" 아이러니가 최근 글과 연속 반복
4. `~더라고요` 발견 어미가 한 글에 2회 이상
5. 마무리 장치 2개 겹침 (예: 괄호 독백 + 다음 액션 선언)
6. `사건 → 겸손 고백("~조차 몰랐다") → 소박한 다음 액션` 아크가 최근 글과 연속 반복 (아크 자체는 좋지만 매 글이면 골격 고정)
7. 수사적 대구 2연속 ("X가 아니라 Y" 프레임 포함) — 드라마틱 조탁으로 읽힌다
8. 격언(시제 없는 일반 명제)이 글 어디든 있음 — 마무리에서만 금지가 아니라 **위치 불문 금지**. 오프닝·중간 규범 문장("~해야 한다")으로 자리를 옮겨 살아남는 게 흔한 우회다

### 후보 모드 ("옵션 여러 개 줘" 요청 시)

사용자가 "옵션 여러 개", "후보로 줘", "몇 개 뽑아줘"라고 하거나 복수 후보를 원하는 신호를 주면 이 모드로 동작한다.

1. **후보 3~4개**를 만들되, 후보마다 **서로 다른 각도를 자동 배정**한다 (사건형/판단형/반성형/관찰형/대화형/숫자형 중 시드에 맞는 3~4개 선택). 첫 줄 골격과 마무리 장치도 후보 간에 겹치지 않게 한다 — 같은 글의 길이 차이가 아니라 각도가 다른 별개의 글이어야 한다.
2. 모든 후보에 **발행 전 검수(§3 STEP 1~3)를 자가 실행**하고, 금지 골격에 걸린 후보는 1회 재작성. 그래도 걸리면 그 후보는 버리고 다른 각도로 교체한다.
3. 출력 포맷 — 후보마다:
   ```
   ### 후보 N [각도: 숫자형]
   (본문)
   > 검수: 통과 / 첫 줄=숫자 대비, 마무리=완료 보고
   ```
4. 사용자가 픽하면, **떨어진 후보의 반려 사유를 한 줄씩** 확인해 기록한다 (예: "첫 줄이 격언 같음"). 이 기록이 쌓이면 이 문서의 금지 골격·각도 가중치를 데이터로 갱신한다.

단일 글 요청(기본)은 이 모드가 아니라 STEP 0부터의 기본 흐름을 따른다.

## 2. 톤 레퍼런스 (실제 글 기반)

### 브랜드 톤 원칙 (BRAND.md 기반)

- **페르소나**: 조금 더 아는 옆자리 선배
- **말투**: 부드러운 존댓말 (~해보세요, ~돼요, ~합니다)
- **격식체 금지**: ~하십시오, ~바랍니다 사용 안 함
- **핵심 키워드**: 편안함, 안정감, 명료함
- **짧은 문장**: 한 문장에 한 가지 정보만
- **험블 톤**: 거만·가르치는 단정 금지. 배우는 중인 사람의 공유. 거창한 단어 금지 ("브랜드를 만들 거예요" ✗ → "사람들의 불편을 풀어주는 걸 만들어 보려고요" ✓)

### Threads 문체 상수 (매 글 유지)

| 항목 | 설명 |
|------|------|
| **줄바꿈 리듬** | 2~3줄마다 빈 줄로 호흡 |
| **경험 기반 진정성** | 개인 경험에서 출발, 구체 디테일 포함 |
| **이모지 절제** | 거의 사용하지 않음. 가끔 1개, `ㅎㅎ` 정도 허용 |
| **취향 기준** | 미니멀, 실용성, 소비취향 공유 중 하나가 은근히 보임 |

### 마무리·유머 장치 (옵션 — 글당 최대 1개, 회전해서 사용)

| 장치 | 예시 | 주의 |
|------|------|------|
| 독자 질문 | "편할 것 같지 않나요?" | 격언화 우회 수단으로 쓰지 않기 |
| 다음 액션 선언 | "숫자 한 칸부터 채우러 간다" | `일단 ~부터` 구문이 기본값이 되기 쉬움 — 연속 글 반복 금지 |
| 괄호 안 독백 | "(이렇게 말해놓으면 누가 만들어주겠지..)" | 틱 되기 쉬움 — 예외적으로만 |
| 완료 보고 | "이 글도 방금 손으로 올렸습니다 ㅎㅎ" | 소재와 맞물릴 때만 |
| 참여 유도 | "같이 지켜봐 주세요" | 시리즈 예고 글에서만 |

겸손 고백("~조차 몰랐다", "하나도 모릅니다")도 험블 톤의 장치다 — 매 글 중반 같은 위치에 넣으면 연출로 읽힌다. 위치와 표현을 회전한다.

### 글 유형별 톤 변화

**진지한 공유 (소개글, 비전)**
- 존댓말 일관 유지
- "~싶습니다", "~하려 합니다"
- 큰 비전 → 구체적 실천으로 좁혀감

**인사이트/의견**
- 반말 허용 (~같다, ~해야겠다)
- 개인적 독백 톤
- 짧고 단정적

**제품/서비스 제안**
- 존댓말 + 질문형
- 가치 제안을 질문으로 포장
- 구체적 리스트로 뒷받침

### 톤 예시

**예시 A: 자기소개 (진지한 공유)**
```
우리 같이 생산자가 되어 봅시다
나, 비즈니스, 프로덕트
이 세 가지 레이어로 구조화해 빠르게 앱을 만들고,
피드백을 받으며 세상에 도움이 되는 제품을 만들어가고 싶습니다.
```

**예시 B: 서비스 제안 (가치 제안)**
```
바이브코딩으로 만든 내 앱, 관리하기 쉬우신가요?
운영 경험 7년차 개발자가 스파게티처럼 꼬인 코드를 정리해주고
운영하기 편하게 만들어준다면 도움을 받으시겠나요?
```

**예시 C: 독백/인사이트 (반말 톤)**
```
바이브코딩이 보편화되고 AI 앱 생태계가 커지더라도
인프라 관리 및 운영 기술은 계속 필요할 것 같다
그것들을 잘 해줄 수 있는 사람으로 포지셔닝 해야겠다
```

**예시 D: 도구 제안 (유머 + 리스트)**
```
내가 하루에 만든 작업내용을 모아서 정리해주고
위클리로 리마인드해주고 openclaw에서 관리할 수 있는 도구 어떠신가요?
(이렇게 말해놓으면 누가 만들어주겠지..)
```

> 예시는 **톤 참고용**이다. 문장 골격(첫 줄 구조·마무리 구조)을 복제하면 골격 회전 규칙 위반.

## 3. 발행 전 검수 워크플로우

사용자가 글을 제공하면 아래 순서대로 검수하고, 결과를 한 번에 출력한다: 맞춤법/비문 → 톤 일관성 → 골격/임팩트.

### STEP 1: 맞춤법/비문 검사 (최우선)

문장 단위로 순회하며 아래를 검사한다:

| 항목 | 검사 내용 |
|------|-----------|
| **맞춤법** | 오탈자, 띄어쓰기 오류 |
| **비문** | 주어-서술어 불일치, 조사 오류, 문장 미완성 |
| **중복** | 같은 내용이 반복되는 문장이나 단락 |
| **어색한 표현** | 번역투, 불필요한 피동/사동 |

발견 시 원문과 수정안을 나란히 제시:
```
[맞춤법] "스스로 성장하는" → "스스로 성장하는"
[비문] "코드를 정리해주고 운영하기 편하게 만들어준다면" → 주어가 앞문장과 불일치. "~정리해드리고 ~만들어드린다면"으로 수정 제안
```

### STEP 2: 톤 일관성 검사

위 「2. 톤 레퍼런스」를 기준으로 톤을 검사한다.

**핵심 체크 포인트:**

1. **페르소나 유지**: "조금 더 아는 옆자리 선배" 톤인가
2. **말투 일관성**: 한 글 안에서 존댓말/반말이 의도 없이 섞이지 않는가
3. **격식체 침범**: "~하십시오", "~바랍니다" 같은 격식체가 끼어들지 않았는가
4. **차가움 경계**: 기업 보도자료처럼 차갑지 않은가
5. **험블 톤**: 성과가 시행착오 없이 이력서식으로 나열되지 않았는가. 가르치는 단정·독자 일반화 조언이 없는가
6. **글 유형별 톤 적합성**:
   - 진지한 공유 → 존댓말 일관
   - 인사이트/의견 → 반말 허용
   - 제품 제안 → 존댓말 + 질문형

### STEP 3: 골격/임팩트 검사

독자(생산 의지가 있는 사람들)에게 꽂히는 글인지, 금지 골격에 걸리지 않는지 검사한다:

| 항목 | 기준 |
|------|------|
| **첫 줄 선언** | 첫 줄이 질문·사정 요약·추상 요약이 아니라 선언인가. 스크롤을 멈추게 하는가 |
| **금지 골격** | §1 「금지 골격」 5개 항목에 하나도 걸리지 않는가 |
| **구체성** | 추상적 선언 대신 구체적 상황/숫자가 있는가 |
| **독자 관점** | "내가 뭘 했나" 나열이 아니라 독자가 "나도 그래"라고 느낄 수 있는가 |
| **마무리** | 자기 구체 문장으로 닫혔는가. 마무리 장치는 1개 이하인가 |
| **줄바꿈 리듬** | 2~3줄마다 빈 줄로 호흡이 있는가 |
| **길이 적절성** | Threads 특성상 너무 길지 않은가 (권장: 5~15줄) |

### 출력 포맷

검수 결과를 아래 형식으로 출력한다:

```
## 검수 결과

### 1. 맞춤법/비문
- (발견 사항 또는 "이상 없음")

### 2. 톤
- (발견 사항 또는 "톤 일관성 양호")

### 3. 골격/임팩트
- (발견 사항 또는 "골격·임팩트 양호")

### 총평
(한두 줄 종합 의견)

### 수정안 (수정이 필요한 경우만)
(원문 전체를 수정 반영한 버전 제시)
```

**수정안 원칙:**
- 원문의 의도와 개성을 최대한 살린다
- 톤을 바꾸지 않는다 — 오류만 고친다
- 수정 부분을 **볼드**로 표시한다
- 수정이 없으면 "수정안" 섹션을 생략한다

### 운영 메모 (2026-04-01)
- 참고 링크: `https://www.threads.com/@biggerthanseoul.ai/post/DWjGRlXmtrU`
- 공개된 `threads_post.md` 기반으로 확인한 운영 포인트:
  1. **댓글/대댓글의 깊이**가 가장 강한 신호다. 댓글 수보다 한 스레드 안에서 여러 번 오가는 긴 대화가 더 중요하다.
  2. **초기 반응 속도(게시 후 30~90분)**가 강한 평가 신호다. 올리고 사라지기보다 최소 1시간 정도는 반응을 받는 운영이 유리하다.
  3. **인용/리포스트**는 팔로워 밖으로 직접 확산시키는 신호다. 한 줄 요약, 인용하기 쉬운 포인트, 의견이 갈릴 여지가 있는 문장이 도움이 된다.
  4. **저장/공유**는 조용하지만 강한 수요 신호다. 체크리스트, 정리본, 단계형 팁, 나중에 다시 볼 만한 문장이 유리하다.
  5. **좋아요는 얕은 신호**라서 목표로 삼기보다 부산물로 보는 편이 낫다.
- 실제 적용 원칙:
  - 시리즈형 글은 메인 글 직후 **첫 댓글을 직접 달아** 대화 시작점을 만든다.
  - 올린 뒤 초반 댓글에는 **짧더라도 바로 답글**해 대화를 길게 이어간다.
  - 본문 말미에는 **인용/리포스트되기 쉬운 한 줄 요약**이나 질문을 넣는다. 단, 마무리 장치 1개 규칙(§1)을 넘지 않는 범위에서.
  - 정보형 글은 **저장 가치가 느껴지는 정리 문장**을 남긴다.
  - 단, 사용자의 현재 문체 선호(담백하고 짧고 현실적인 느낌)를 해치지 않는 범위에서만 반영한다.
