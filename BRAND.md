# Sam Samuel 브랜드 아이덴티티

## 핵심 정보

| 항목       | 내용                                                    |
| ---------- | ------------------------------------------------------- |
| **이름**   | Sam Samuel (가칭 — 확정 전)                              |
| **슬로건** | 생산자를 만든다                                         |
| **미션**   | 시작의 장벽을 낮춰서, 만들고 싶은 사람이 만들 수 있게   |
| **비전**   | 모든 사용자가 생산자가 되는 세상                         |

## 왜 Sam Samuel인가

뭔가 만들고 싶은 마음은 있는데, 어디서부터 시작해야 할지 모르겠다. 도구는 넘치는데 첫 발을 떼기가 어렵다. Sam Samuel은 이 **시작의 장벽**을 문제로 본다.

우리가 해결하는 건 기술적 역량의 부족이 아니라, 시작하지 못하게 만드는 막연함이다. 복잡한 도구를 단순하게, 막막한 첫 화면을 친근하게 바꿔서, 사용자가 "일단 해볼까"라고 느끼게 만드는 것. 그게 Sam Samuel이 존재하는 이유다.

## 만드는 것

**사람들이 곤란을 겪고 있는 문제를 해결하는 모든 도구.**

특정 카테고리에 얽매이지 않는다. 누군가 막혀 있으면, 그 막힌 곳을 뚫어주는 도구를 만든다. 형태는 앱이 될 수도 있고, 자동화 워크플로우가 될 수도 있고, 단순한 유틸리티가 될 수도 있다.

판단 기준은 하나다: **이걸 쓰면 곤란이 해결되는가.**

## 대상

**생산 의지가 있는 모든 사람.** 직업, 규모, 경험 수준을 가리지 않는다.

- 사이드 프로젝트를 시작하고 싶은 직장인
- 아이디어는 있지만 구현 방법을 모르는 비개발자
- 새로운 도구를 배우며 성장하고 싶은 주니어
- 반복 업무를 자동화하고 싶은 1인 운영자

이들의 공통점은 하나다: **만들고 싶은 마음이 이미 있다.** Sam Samuel은 그 마음에 도구를 더해준다.

## 핵심 가치

세 가지는 동등하다. 어느 하나가 다른 것을 압도하지 않는다. 셋의 균형이 Sam Samuel다움이다.

1. **시작을 쉽게** — 첫 화면에서 뭘 해야 하는지 바로 알 수 있어야 한다
2. **스스로 성장하는 환경** — 가르치는 게 아니라, 써보면서 알게 되는 구조를 만든다
3. **최소한의 도구, 최대한의 가능성** — 기능을 쌓는 게 아니라 제약을 줄인다

## 브랜드 성격

Sam Samuel은 **조금 더 아는 옆자리 선배**다.

경험이 있어서 팁을 줄 수 있지만, 가르치려 들지 않는다. "나도 처음에 헤맸는데"라고 말할 수 있는 사람. 사용자보다 약간 앞서 있되, 그 간격을 권위로 쓰지 않는다.

- "아, 그거? 이렇게 하면 돼요" — 아는 걸 가볍게 알려준다
- "이거 먼저 해보는 게 나을 거예요" — 경험에서 나온 순서를 제안한다
- 사용자를 초보자 취급하지 않되, 어려운 것을 쉬운 말로 설명한다
- 완성된 결과물보다 만드는 과정의 동반자 역할

> 비유하면: 카페에서 노트북 펴놓고 옆자리에서 같이 작업하는 사람. 물어보면 바로 답해주고, 안 물어보면 조용히 자기 일 한다.

## 감정적 약속

사용자가 Sam Samuel 제품을 쓴 뒤 느껴야 할 감정: **"계속 쓰고 싶다."**

화려한 첫인상이 아니라, 조용히 습관이 되는 도구. 한 번 쓰고 감탄하는 게 아니라, 매일 열게 되는 편안함. 그래서 우리 디자인의 모든 판단은 "이게 매일 쓰기에 불편하지 않은가?"로 귀결된다.

따뜻함이 가장 먼저 느껴져야 하는 순간은 **첫 사용**이다. 처음 열었을 때 막막하지 않고, 환영받는 느낌. 이 첫인상이 "계속 쓰고 싶다"로 이어지는 출발점이다. 첫 화면에서 차갑거나 복잡하면, 두 번째 방문은 없다.

## Sam Samuel이 아닌 것

브랜드가 명확하려면, 되지 않을 것도 명확해야 한다.

| 우리는 이것이 **아니다**       | 왜                                                       |
| ------------------------------ | -------------------------------------------------------- |
| 전문가 전용 도구               | 진입장벽을 높이는 순간 우리 미션과 모순된다               |
| 기능 과잉 플랫폼               | 뭐든 다 되는 도구는 아무것도 잘 못한다                    |
| 트렌디한 장난감                | 유행을 따르면 유행과 함께 사라진다                        |
| **차가운 기업 도구** (최대 경계) | 효율만 남기면 사람이 빠진다. 따뜻함이 우리의 차별점이다   |

> **가장 빠지기 쉬운 함정은 차가워지는 것이다.** 최적화하고, 정리하고, 효율을 높이다 보면 따뜻함이 사라진다. 기능을 추가할 때, UI를 다듬을 때, 문구를 쓸 때 항상 "이게 차갑지 않은가?"를 점검한다.

## 톤앤매너

### 말투: 존댓말 부드러운 체

격식체가 아니라, 부드러운 존댓말을 쓴다. "~하십시오"가 아니라 "~해보세요", "~돼요".

| 상황         | 이렇게 (O)                                    | 이렇게 안 함 (X)                               |
| ------------ | --------------------------------------------- | ---------------------------------------------- |
| 행동 유도    | "시작해볼까요? 여기를 눌러보세요."              | "시작하십시오."                                 |
| 입력 요청    | "이름만 입력하면 돼요."                         | "이름을 입력해 주십시오."                        |
| 안내         | "천천히 따라해보세요."                           | "아래 절차를 수행하세요."                        |
| 에러         | "이런, 문제가 생겼어요. 다시 해볼까요?"          | "오류가 발생했습니다. 재시도하십시오."             |

### 원칙

| 키워드       | 의미                                          | 적용                                           |
| ------------ | --------------------------------------------- | ---------------------------------------------- |
| **편안함**   | 부담 없는 말투, 부드러운 존댓말                 | 위 말투 표 참조                                 |
| **안정감**   | 급하지 않은 리듬, 여유 있는 여백               | 한 화면에 한 가지 행동만 요구                   |
| **명료함**   | 모호한 설명 없이 한 문장으로 이해              | "여기에 이름을 입력하면 프로필이 생성돼요"        |

### 글쓰기 가이드

- 짧은 문장을 쓴다. 한 문장에 한 가지 정보만 담는다.
- 기능 설명 시 "왜"를 먼저, "어떻게"를 나중에 쓴다.
- 사용자에게 입력을 요구할 때는 그 이유가 흐름 안에서 자연스럽게 드러나야 한다.
- 에러 메시지도 친근하게. 원인과 해결을 같이 알려준다.
- "~하십시오", "~바랍니다" 같은 격식체는 쓰지 않는다.

## 디자인

### 컨셉

**미니멀하고 젠 스타일.** 타이포그래피 위주로, 기능 자체가 디자인으로 설명되어야 한다.

디자인의 모든 판단은 세 가지 질문으로 귀결된다:
1. **이게 따뜻한가?** — 차가워지는 것이 가장 큰 위험이다 (안티 포지셔닝 참조)
2. **이게 자연스러운가?** — 인공적이고 기계적인 느낌을 경계한다
3. **매일 쓰기에 불편하지 않은가?** — 습관이 되는 편안함이 목표다

### 자연의 언어

Sam Samuel의 디자인은 자연에서 가져온다. 분명하게 느낄 수 있는 수준으로. 디자인 원천이 되는 세 가지 이미지: **아침 햇살, 나무와 숲, 물과 흐름.**

가장 중요한 요소는 **곡선과 형태**다. 그 다음이 색감, 여백 순서.

#### 1. 곡선/형태 (최우선)

| O (이렇게)                                    | X (이렇게 안 함)                              |
| --------------------------------------------- | --------------------------------------------- |
| 모서리에 부드러운 라운딩                        | 직각 모서리                                    |
| 카드마다 미세하게 다른 곡률                      | 모든 요소에 동일한 border-radius               |
| 유기적인 둥근 형태 (조약돌, 잎사귀)              | 완벽한 정원, 정사각형                           |
| 구분선 대신 여백과 곡면으로 영역 분리             | 직선 구분선(hr)으로 영역 자르기                 |
| 아이콘도 둥글고 부드러운 라인                    | 날카로운 각진 아이콘                           |

#### 2. 색감

| O (이렇게)                                    | X (이렇게 안 함)                              |
| --------------------------------------------- | --------------------------------------------- |
| 배경에 미세한 warm 그라데이션                   | 순수 하얀 `#FFFFFF` 배경                       |
| 자연에서 뽑은 톤 (따뜻한 베이지, 크림, 올리브)   | 차가운 순색 (순수 파랑, 순수 회색)              |
| 카드 배경에도 미묘한 톤 차이                     | 모든 카드 동일한 배경색                         |
| 다크모드도 따뜻한 차콜 베이스                    | 순수 검정 `#000000` 다크모드                    |

#### 3. 여백/공간

| O (이렇게)                                    | X (이렇게 안 함)                              |
| --------------------------------------------- | --------------------------------------------- |
| 콘텐츠 사이에 숨 쉴 수 있는 넉넉한 간격          | 빈틈없이 요소로 꽉 찬 화면                     |
| 밀집 영역과 트인 영역이 교차하는 리듬             | 모든 간격이 균일한 기계적 배치                  |
| 화면 가장자리에 충분한 패딩                      | 콘텐츠가 화면 끝까지 붙어 있음                  |
| 중요한 요소 주변에 더 넓은 여백                   | 위계 없이 모든 요소에 같은 여백                 |

### 레퍼런스

| 레퍼런스       | 취하는 것                                                    |
| -------------- | ------------------------------------------------------------ |
| **Braun**      | 잘 만들어진 도구의 촉감. 물리적 피드백, 정제된 미니멀리즘      |
| **교보문고**   | 따뜻한 공간감. 들어서면 머물고 싶어지는 서재의 분위기          |
| **츠타야**     | 큐레이션된 경험. 물건이 아니라 맥락을 보여주는 배치            |

공통점: 세 곳 모두 **머물고 싶은 곳**이다. Sam Samuel의 제품도 "빨리 끝내고 나가는 도구"가 아니라 **"머물며 만드는 공간"**이어야 한다.

### 브랜드 텍스처: Warm Tool

Sam Samuel의 시각적 서명은 **"따뜻한 도구"**. 미니멀하지만 밋밋하지 않고, 모든 요소가 물리적 촉감을 가진다.

| 요소         | 규칙                                                        |
| ------------ | ----------------------------------------------------------- |
| **그림자**   | 순수 검정 금지. 따뜻한 톤으로 조명처럼 자연스러운 깊이 표현   |
| **버튼**     | "누를 수 있는" 물리적 피드백. 살짝 볼록하거나 눌리는 느낌     |
| **입력 필드** | "오목한" 리세스 느낌. 글을 쓸 수 있는 공간이라는 인지          |
| **곡률**     | 둥글되, AI 서비스에 흔한 과도한 곡선은 피한다                 |
| **그라데이션** | 원근감과 입체감 표현. 평면이 아닌 깊이 있는 공간감            |

> 상세 구현: [DESIGN_SYSTEM.md](./DESIGN_SYSTEM.md) "브랜드 텍스처: Warm Tool" 참조

### 컬러

- **기본 모드는 라이트모드.** 밝고 따뜻한 첫인상이 브랜드의 출발점이다.
- 다크모드도 지원하되, 완전 검은색은 피한다 — 따뜻한 차콜 기반.
- 키 컬러는 각 앱의 특성을 참고해서 하나를 뽑는다.
- 흰색과 검은색을 메인으로, 채도와 명도 조절로 위계를 설정한다.

### 타이포그래피

- Pretendard 단일 폰트. 굵기와 크기만으로 의도를 전달한다.
- 폰트 하나로 제한하는 이유: 위계는 서체가 아니라 크기와 굵기로 만든다.

### 모션

**감성적 모션.** 사용자 행동에 반응하는 즐거운 마이크로 애니메이션을 적용한다.

| 상황           | 모션 방향                                                   |
| -------------- | ----------------------------------------------------------- |
| **화면 전환**  | 부드러운 fade/slide. 갑작스러운 전환 금지                    |
| **요소 등장**  | 자연스럽게 나타남. 튕기거나 과장된 바운스 없이               |
| **호버/프레스** | 미묘한 반응. 버튼이 살짝 눌리거나 카드가 살짝 떠오르는 정도  |
| **성공 피드백** | 작은 기쁨을 주는 모션. 완료 체크, 저장 확인 등               |
| **로딩**       | 기다림이 지루하지 않은 부드러운 전환                         |

원칙: 모션은 **기다려주는 느낌**을 줘야 한다. 빠르고 효율적인 것보다 부드럽고 편안한 것이 우선. 단, 사용자를 기다리게 하는 불필요한 지연은 금지.

### 접근성

**WCAG AAA를 목표로 한다.**

- 대비비 7:1 이상 (텍스트와 배경)
- 키보드 네비게이션 완벽 지원
- 스크린리더 호환
- 추가 시각적 단서 제공 (색만으로 정보를 구분하지 않음)
- 포커스 인디케이터 명확하게

> "모든 사용자가 생산자가 된다"는 비전에서 "모든"이 진짜 모든이려면, 접근성은 선택이 아니다.

### 반응형 전략

- 모바일 우선하되 데스크탑에서도 어색하지 않도록
  - Desktop: 사이드바(260px) + 메인 콘텐츠
  - Mobile: 사이드바 제거 → 하단 플로팅 탭바 전환, 카드 그리드 축소
- 같은 콘텐츠가 디바이스에 따라 자연스럽게 재배치되어야 한다

### 레이아웃 원칙

정보의 **위계**가 모든 디자인 판단의 기준이다. 위계를 만드는 수단:

- 크기, 컬러, 여백, 질감, 뎁스(원근/아웃포커싱), 움직임
- 리듬 — 크게 → 작게 → 크게 식의 시각적 완급 조절

설계 순서: 컨셉 → 설계 → 구도 → 배치 → 색감

#### 참고하는 디자인 이론

- **로빈 윌리엄스 4원칙**: 근접성(그룹화), 정렬(그리드), 반복(일관성), 대비(위계)
- **시선 유도**: F-패턴(좌→우, 위→아래), Z-패턴(훑어보기)
- **게슈탈트 심리학**: 인간이 시각 정보를 그룹화하여 인식하는 원리 활용

### 핵심 철학

의도적으로 최소한만 만든다. 그 외는 만들지 않는다.

1. **컬러 토큰** — 의미 중심 (semantic). "이 색은 언제 쓰는가"가 드러나야 함
2. **타이포 + 간격** — 4단계 스케일, 1개 폰트, spacing 토큰만 사용
3. **핵심 컴포넌트 8종** — Button, Input, Select, Modal, Toast, Badge, InputGroup, Card
4. **사용 규칙** — 문서가 아니라 제약으로 강제

### 절대 하지 않는 것

- blue-100 ~ blue-900 팔레트 중심 토큰
- ghost, outline, subtle, link 등 과잉 변형
- 컴포넌트마다 제각각 margin
